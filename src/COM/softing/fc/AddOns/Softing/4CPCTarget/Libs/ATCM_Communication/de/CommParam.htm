Verbindungsparameter

Parameter zur Definition der Eigenschaften einer neuen Verbindung.

Syntax

Einzelne Parameter werden durch Komma vorneinander getrennt. Die Bedeutung der Paramater wird durch ihre Reihenfolge im Verbindungsstring definiert:
<Port nb>,<Baudrate>,<Data bits>,<Parity>,<Stop bits>,<Protocol>

Beispiel: Verbindungsstring := "1,9600,8,N,1,NO"

Bedeutung der Parameter

Parameter Bedeutung Mögliche Werte
<Port nb> Nummer des COM-Ports 1 - 9
<Baudrate> Baudrate 110, 300, 600, 1200, 2400, 4800, 9600, 14400, 19200, 38400, 56000, 57600, 115200, 128000, 230400, 256000, 460800, 921600
<Data bits> Anzahl der Bits je Datum 4, 5, 6, 7, 8
<Parity> Parität N kein
E gerade
O ungerade
M Markierung
S Leerzeichen
<Stop bits> Anzahl Stopbits 1, 1.5, 2
<Protocol> Verwendetes Protokoll NO    kein Protokoll
SW     Xon/Xoff (Software-Handshake)
HW    RTS/CTS (Hardware-Handshake)

Falls ein Parameter fehlt, werden die folgenden Standardwerte verwendet:

  • Port nb:1
  • Baudrate: 9600
  • Databits: 8
  • Parity: N
  • Stopbits: 1
  • Protocol: NO  

Beschreibung

Die Verbindungsparameter im Verbindungsstring, werden an den Parameter sParam einer Instanz der Struktur COMM_PARTNER übergeben. In ihnen werden zentrale Eigenschaften für eine mit CONNECT hergestellte serielle Verbindung festgelegt.
