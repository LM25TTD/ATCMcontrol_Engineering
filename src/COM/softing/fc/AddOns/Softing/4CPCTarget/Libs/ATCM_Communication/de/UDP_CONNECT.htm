UDP_CONNECT

Initialisiert oder deaktiviert den durch einen Socket repräsentierten Kommunikationsendpunkt.

Schnittstelle

FUNCTION_BLOCK UDP_CONNECT

VAR_INPUT
    EN_C : BOOL;           
END_VAR

VAR_IN_OUT
    SOCKET_HANDLE : UDP_SOCKET_HANDLE; 
END_VAR

VAR_OUTPUT
    VALID        : BOOL;   
    ERROR        : BOOL;   
    ERROR_NR     : DINT;
    ERROR_STRING : WSTRING;
END_VAR

Parameter

INPUT

EN_C
Wert vom Typ BOOL.
Spezifiziert, ob der Socket initialisiert oder deaktiviert werden soll.
Mögliche Werte:
TRUE: Socket wird initialisiert.
FALSE: Socket wird deaktiviert.

IN_OUT

SOCKET_HANDLE
Instanz der Struktur UDP_SOCKET_HANDLE.
Spezifiziert den Socket, der initialisiert oder deaktivert werden soll.

OUTPUT

VALID
Wert vom Typ BOOL.
Signalisiert, ob der Socket initialisiert oder deaktiviert ist.
Mögliche Werte:
TRUE: Socket ist initialisiert.
FALSE: Socket ist deaktiviert.
ERROR
Wert vom Typ BOOL.
Signalisiert den Fehlerstatus.
Mögliche Werte:
TRUE: Ein Fehler ist aufgetreten.
FALSE: Es ist kein Fehler aufgetreten.
ERROR_NR
Wert vom Typ DINT.
Enthält eine Fehlernummer, falls ein Fehler aufgetreten ist.
Ist kein Fehler aufgetreten, hat diese Variable den Wert 0.
Mögliche Werte sind in den UDP-Fehlerkonstanten definiert.
ERROR_STRING
Wert vom Typ WSTRING.
Enthält einen Fehlertext, falls ein Fehler aufgetreten ist.
Ist kein Fehler aufgetreten, enthält diese Variable einen Leerstring.

Beschreibung

Der Aufruf einer Instanz von UDP_CONNECT initialisiert oder deaktiviert den an SOCKET_HANDLE übergebenen Socket. Der Socket wird durch eine Instanz von UDP_SOCKET_HANDLE repräsentiert. Vor der Initialisierung muß dem Element LocalPort der übergebenen Instanz eine Portnummer zugewiesen werden, um festzulegen, an welcher Adresse der Socket Botschaften empfängt. Nur über einen initialisierten Socket können Botschaften empfangen und gesendet werden. Ob der Socket aktuell initalisiert ist, kann über den Parameter VALID von UDP_CONNECT überprüft werden.
