COPY_WSTRING_FROM_BYTE_STRING

Reads a defined number of characters from a determined position within a bytestring, converts them and copies the bytes read into a WSTRING.

Interface definition

FUNCTION COPY_WSTRING_FROM_BYTE_STRING : WSTRING
    VAR_INPUT
        NUM : DINT;
    END_VAR
    VAR_IN_OUT
        POS : DINT;
        BSTR : BYTE_STRING;
    END_VAR

Return value

Value of type WSTRING
Value that has been read out of the bytestring.

Parameters

INPUT

NUM
Value of type DINT
Specifies the number of the characters to be returned in the return value.

IN_OUT

POS
Value of type DINT
Specifies the position in the bytestring from which the value is to be read. Counting starts from 0 on for the first position within the bytestring and is carried out in the unit bytes.
After the function has been carried out successfully, POS is overwritten with the value of the first position in the bytestring that follows the last byte that has been read. If the operation was a failure, POS has the value -1.
BSTR
Instance of the structure BYTE_STRING
Bytestring from which a value is to be read.

Description

COPY_WSTRING_FROM_BYTE_STRING reads a value of type WSTRING from the position passed in POS on from the bytestring passed in BSTR, converts and copies the bytes read into a WSTRING, and returns it as return value. The number of characters that are returned by the function in this WSTRING is defined in the parameter NUM.

In contrast to the function BYTE_STRING_TO_WSTRING, which is also included in the 4C_Communication library, COPY_WSTRING_FROM_BYTE_STRING supports the correct interpretation of umlauts. For this reason, use COPY_WSTRING_FROM_BYTE_STRING in order to read strings from textfiles with function blocks of the 4C_FileAccess library. On the other side, the function BYTE_STRING_TO_WSTRING should be used for reading bytestrings at serial communication and communication via UDP.
