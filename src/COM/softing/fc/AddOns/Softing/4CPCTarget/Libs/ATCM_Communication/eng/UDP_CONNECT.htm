UDP_CONNECT

Initializes or deactivates a socket.

Interface definition

FUNCTION_BLOCK UDP_CONNECT

VAR_INPUT
    EN_C : BOOL;           
END_VAR

VAR_IN_OUT
    SOCKET_HANDLE : UDP_SOCKET_HANDLE; 
END_VAR

VAR_OUTPUT
    VALID        : BOOL;   
    ERROR        : BOOL;   
    ERROR_NR     : DINT;
    ERROR_STRING : WSTRING;
END_VAR

Parameters

INPUT

EN_C
Value of type BOOL.
Specifies whether the socket is to be initialized or deactivated.
Possible values:
TRUE: Socket becomes initialized.
FALSE: Socket becomes deactivated.

IN_OUT

SOCKET_HANDLE
Instance of the structure UDP_SOCKET_HANDLE.
Specifies the socket to be initialized or deactivated.

OUTPUT

VALID
Value of type BOOL.
Signals whether the socket is initialized or deactivated.
Possible values:
TRUE: Socket is initialized.
FALSE: Socket is deactivated.
ERROR
Value of type BOOL.
Signals the error state.
Possible values:
TRUE: An error occurred.
FALSE: No error occurred.
ERROR_NR
Value of type DINT.
Contains an error number if an error occurred.
If no error ocurred, this variable has the value 0.
Possible values are defined within the UDP-error constants.
ERROR_STRING
Value of type WSTRING.
Contains an error text in case of an error.
this variable contains an empty string if no error occurred.

Description

The call of an instance of UDP_CONNECT initializes or deactivates the socket passed to SOCKET_HANDLE. The socket is represented by an instance of UDP_SOCKET_HANDLE. In order to determine at which address messages are received by the socket, a port number has to be assigned to the element LocalPort of the instance passed before initializing. Messages can only be sent or received via a initialized socket. Whether the socket is currently initialized or not can be checked via the parameters VALID and UDP_CONNECT.
