Zugriffsmodus-Konstanten

Konstanten für den Zugriffsmodus auf eine geöffnete oder neu angelegte Datei.

Konstante Numerischer Wert Dateizeiger-Position Beschreibung
CFA_READ 2 Dateianfang
0 = CFA_BOF		 Datei wird für Nur-Lesezugriff geöffnet. Zugriffe auf diese Datei durch andere Tasks oder andere Benutzer sind erlaubt. Wenn die Datei nicht existiert, wird ein Fehler generiert.
CFA_READ_WRITE 6 Dateianfang
0 = CFA_BOF		 Datei wird für Lese- und Schreibzugriff geöffnet. Falls die Datei nicht existiert, wird sie erzeugt. Zugriffe auf diese Datei durch andere Tasks oder andere Benutzer sind erlaubt.
CFA_APPEND 7 Dateiende
-1 = CFA_EOF		 Datei wird für Lese- und Schreibzugriff geöffnet. Falls die Datei nicht existiert, wird sie erzeugt. Zugriffe auf diese Datei durch andere Tasks oder andere Benutzer sind erlaubt.

Beschreibung

Die Zugriffsmodus-Konstanten werden dem Parameter MODE der Strukturen FA_ASYNC_FILE oder FA_SYNC_FILE, die jeweils ein Dateiobjekt repräsentieren, zugewiesen. Dieser Parameter muß immer vor der Initialisierung eines Zugriffszyklus mit FA_ASYNC_OPENFILE oder FA_SYNC_OPENFILE für das an diese Bausteine übergebene Dateiobjekt gesetzt werden.

Mittels des MODE-Parameters wird die Zugriffsberechtigung auf eine Datei festgelegt. Wurde bspw. der Parameter MODE eines Dateiobjekts mit dem Wert CFA_READ vor der Übergabe an FA_ASYNC_OPENFILE initialisiert, kann auf dieses Dateiobjekt nur lesend zugegriffen werden. Schreibzugriffe lösen dagegen einen Fehler aus.

Ein in MODE gesetzter Zugriffsmodus ist nur dann wirksam, wenn er vor der Übergabe an FA_ASYNC_OPENFILE oder FA_SYNC_OPENFILE für das übergebene Dateiobjekt gesetzt wurde. Ein bei der Initialisierung eines Zugriffszyklus gesetzter Zugriffsmodus ist für die Dauer eines kompletten Zugriffszyklus gültig. Änderungen des Werts von MODE innerhalb eines Zugriffszyklus haben daher keine Auswirkung. Um nach dem Lesen einer mit CFA_READ für Nur-Lesezugriff geöffneten Datei schreibend auf diese zuzugreifen, muß die Datei daher erst geschlossen und nach dem Ändern des MODE-Parameters des Dateiobjekts erneut geöffnet werden.

Außer der Zugriffsberechtigung definiert der Zugriffsmodus auch die Position des Dateizeigers nach dem Öffnen der Datei.

Den Zugriffsmodus-Konstanten CFA_READ und CFA_READ_WRITE ist der Wert CFA_BOF zugeordnet. Dieser Wert bewirkt, daß der Dateizeiger beim Öffnen einer Datei immer an den Dateianfang positioniert wird.

Der Zugriffsmodus-Konstante CFA_APPEND ist der Wert CFA_EOF zugeordnet. Dieser Wert bewirkt, daß der Dateizeiger beim Öffnen einer Datei immer an das Dateiende positioniert wird. Ein direkt nach dem Öffnen erfolgender Schreibzugriff hängt daher die geschriebenen Daten immer an das Dateiende an, ohne bereits vorhandene Daten zu überschreiben.

Die beim Öffnen einer Datei mit der Definition des Zugriffsmodus gesetzte Dateizeiger-Position überschreibt den Wert des Strukturelements POSITION einer Strukturinstanz von FA_ASYNC_FILE bzw. FA_SYNC_FILE. Nach dem Öffnen einer Datei steht daher der Dateizeiger immer auf der Position, die durch den Zugriffsmodus definiert ist, unabhängig von dem Wert, der für POSITION in der an den öffnenden Baustein übergebenen Strukturinstanz gesetzt wurde. Soll eine Dateioperation explizit an einer bestimmten Dateiposition erfolgen, muß daher nach dem Öffnen einer Datei der Wert von POSITION neu gesetzt werden.
