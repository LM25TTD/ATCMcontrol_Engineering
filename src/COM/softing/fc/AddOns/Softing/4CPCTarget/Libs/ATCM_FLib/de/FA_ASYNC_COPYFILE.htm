FA_ASYNC_COPYFILE

Kopiert eine Datei in das angegebene Zielverzeichnis.

Schnittstelle

FUNCTION_BLOCK FA_ASYNC_COPYFILE

VAR_INPUT
    EXEC        : BOOL;             
    DESTNAME    : WSTRING;          
    OVERWRITE   : BOOL := TRUE;     
END_VAR
VAR_IN_OUT
    FILE        : FA_ASYNC_FILE;    
END_VAR
VAR_OUTPUT
    DONE        : BOOL;             
    ERROR       : BOOL;             
    ERRCD       : DINT;             
    ERRTXT      : WSTRING;          
END_VAR

Parameter

INPUT

EXEC
Wert vom Typ BOOL
Startet die Operation bei steigender Flanke.
DESTNAME
Wert vom Typ WSTRING
Spezifiziert das Zielverzeichnis und den Namen, den die kopierte Datei erhalten soll. Wird kein Zielverzeichnis angegeben, wird die Datei unter dem spezifizierten Namen in das Standard-Arbeitsverzeichnis von ATCMControl kopiert. Dieses Verzeichnis hat den Namen data und befindet sich direkt unter dem ATCMControl-Installationsverzeichnis.
OVERWRITE
Wert vom Typ BOOL
Legt fest, ob eine Datei überschrieben wird, wenn sie bereits existiert.
Mögliche Werte:
TRUE: Eine bereits existierende Datei wird überschrieben.
FALSE: Eine bereits existierende Datei wird nicht überschrieben. Der Vorgang wird abgebrochen, falls eine Datei gleichen Namens im angegebenen Zielverzeichnis bereits existiert.
Standardwert ist TRUE.

IN_OUT

FILE
Instanz der Struktur FA_ASYNC_FILE
Spezifiziert die Datei, die kopiert werden soll.

OUTPUT

DONE
Wert vom Typ BOOL
Signalisiert den (erfolgreichen oder gescheiterten) Abschluß der Operation.
Mögliche Werte:
TRUE: Operation ist abgeschlossen.
FALSE: Operation ist noch nicht abgeschlossen.
ERROR
Wert vom Typ BOOL
Signalisiert den Fehlerstatus.
Mögliche Werte:
TRUE: Ein Fehler ist aufgetreten.
FALSE: Es ist kein Fehler aufgetreten.
ERRCD
Wert vom Typ DINT
Enthält eine Fehlernummer, falls ein Fehler aufgetreten ist.
Ist kein Fehler aufgetreten, hat ERRCD den Wert 0.
Mögliche Werte sind in den Fehlerkonstanten definiert.
ERRTXT
Wert vom Typ WSTRING
Enthält einen Fehlertext, falls ein Fehler aufgetreten ist.
Ist kein Fehler aufgetreten, enthält ERRTXT einen Leerstring.

Beschreibung

FA_ASYNC_COPYFILE kopiert die im Parameter FILE angegebene Datei in das in DESTNAME spezifizierte Zielverzeichnis und benennt die Datei gegebenenfalls um. Die Quelldatei muß vor dem Kopieren nicht geöffnet werden, d.h. die Ausführung von FA_ASYNC_COPYFILE ist unabhängig von einem Zugriffszyklus.
