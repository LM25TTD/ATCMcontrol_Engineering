FA_ASYNC_CREATEDIRECTORY

Erzeugt ein neues Verzeichnis.

Schnittstelle

FUNCTION_BLOCK FA_ASYNC_CREATEDIRECTORY
VAR_INPUT
    EXEC        : BOOL;             
END_VAR
VAR_IN_OUT
    FILE        : FA_ASYNC_FILE;    
END_VAR
VAR_OUTPUT
    DONE        : BOOL;             
    ERROR       : BOOL;
    ERRCD       : DINT;
    ERRTXT      : WSTRING;
END_VAR

Parameter

INPUT

EXEC
Wert vom Typ BOOL
Startet die Operation bei steigender Flanke

IN_OUT

FILE
Instanz der Struktur FA_ASYNC_FILE
Im Gegensatz zu den anderen Funktionen der Bausteingruppe für elementare asynchrone Dateioperationen repräsentiert die an FILE übergebene Strukturinstanz nicht ein Dateiobjekt, auf das die betreffenden Dateioperation ausgeführt wird. Stattdessen wird das Element FILENAME der Struktur FA_ASYNC_FILE für die Definition des Verzeichnisnamens genutzt.

OUTPUT

DONE
Wert vom Typ BOOL
Signalisiert den (erfolgreichen oder gescheiterten) Abschluß der Operation.
Mögliche Werte:
TRUE: Operation ist abgeschlossen.
FALSE: Operation ist noch nicht abgeschlossen.
ERROR
Wert vom Typ BOOL
Signalisiert den Fehlerstatus.
Mögliche Werte:
TRUE: Ein Fehler ist aufgetreten.
FALSE: Es ist kein Fehler aufgetreten.
ERRCD
Wert vom Typ DINT
Enthält eine Fehlernummer, falls ein Fehler aufgetreten ist.
Ist kein Fehler aufgetreten, hat ERRCD den Wert 0.
Mögliche Werte sind in den Fehlerkonstanten definiert.
ERRTXT
Wert vom Typ WSTRING
Enthält einen Fehlertext, falls ein Fehler aufgetreten ist.
Ist kein Fehler aufgetreten, enthält ERRTXT einen Leerstring.

Beschreibung

Mit FA_ASYNC_CREATEDIRECTORY wird ein neues Verzeichnis angelegt. Eine neue Datei kann mit einer Instanz des Funktionsbausteins FA_ASYNC_OPENFILE nur in einem bereits existierenden Verzeichnis generiert werden. Verwenden Sie eine Instanz von FA_ASYNC_CREATEDIRECTORY, wenn Sie eine neue Datei in einem noch nicht existierenden Verzeichnis generieren wollen, um dieses Verzeichnis zu erzeugen.

Der Name des neu anzulegenden Verzeichnisses wird FA_ASYNC_CREATEDIRECTORY über den Parameter FILE übergeben. FILE erwartet eine Instanz der Struktur FA_ASYNC_FILE. Diese Struktur enthält das Strukturelement FILENAME, das einen Verzeichnis- oder Dateinamen spezifiziert und vor Übergabe der Strukturinstanz an FA_ASYNC_CREATEDIRECTORY initialisiert werden muß.

Bei der Initialisierung des Strukturelements FILENAME für das Erzeugen eines neuen Verzeichnisses darf nur das Verzeichnis ohne Dateinamen angegeben werden. Der Pfad, in dem das Verzeichnis angelegt werden soll, muß voll qualifiziert sein (z.B. C:\Projekte). Die korrekte Syntax von Pfad- und Verzeichnisangabe wird vom Compiler nicht überprüft. Existiert das Verzeichnis bereits, oder ist die Pfad- oder Verzechnisangabe nicht korrekt, wird zur Laufzeit ein Fehler ausgelöst.
