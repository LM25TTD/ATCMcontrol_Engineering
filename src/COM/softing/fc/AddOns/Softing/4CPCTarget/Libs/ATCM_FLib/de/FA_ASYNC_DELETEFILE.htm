FA_ASYNC_DELETEFILE

Löscht eine Datei oder ein Verzeichnis.

Schnittstelle

FUNCTION_BLOCK FA_ASYNC_DELETEFILE
VAR_INPUT
    EXEC        : BOOL;             
END_VAR
VAR_IN_OUT
    FILE        : FA_ASYNC_FILE;    
END_VAR
VAR_OUTPUT
    DONE        : BOOL;             
    ERROR       : BOOL;
    ERRCD       : DINT;
    ERRTXT      : WSTRING;
END_VAR

Parameter

INPUT

EXEC
Wert vom Typ BOOL
Startet die Operation bei steigender Flanke.

IN_OUT

FILE
Instanz der Struktur FA_ASYNC_FILE
Dateiobjekt, auf das die Operation ausgeführt wird. Die zu löschende Datei wird im Parameter FILENAME der übergebenen Instanz von FA_ASYNC_FILE spezifiziert.
Statt einer Datei kann auch ein Verzeichnis gelöscht werden. Zu diesem Zweck wird im Parameter FILENAME der übergebenen Instanz von FA_ASYNC_FILE nur ein Verzeichnispfad ohne Dateinamen spezifiziert, z.B. "D:\Projekte\Testprojekt". Es können nur leere und nicht schreibgeschützte Verzeichnisse gelöscht werden. Ist das spezifizierte Verzeichnis nicht leer oder schreibgeschützt, wird ein Fehler generiert.

OUTPUT

DONE
Wert vom Typ BOOL
Signalisiert den (erfolgreichen oder gescheiterten) Abschluß der Operation.
Mögliche Werte:
TRUE: Operation ist abgeschlossen.
FALSE: Operation ist noch nicht abgeschlossen.
ERROR
Wert vom Typ BOOL
Signalisiert den Fehlerstatus.
Mögliche Werte:
TRUE: Ein Fehler ist aufgetreten.
FALSE: Es ist kein Fehler aufgetreten.
ERRCD
Wert vom Typ DINT
Enthält eine Fehlernummer, falls ein Fehler aufgetreten ist.
Ist kein Fehler aufgetreten, hat ERRCD den Wert 0.
Mögliche Werte sind in den Fehlerkonstanten definiert.
ERRTXT
Wert vom Typ WSTRING
Enthält einen Fehlertext, falls ein Fehler aufgetreten ist.
Ist kein Fehler aufgetreten, enthält ERRTXT einen Leerstring.

Beschreibung

Löscht die an FILE übergebene Datei. Die zu löschende Datei wird im Parameter FILENAME der an FILE übergebenen Strukturinstanz von FA_ASYNC_FILE spezifiziert.

FA_ASYNC_DELETEFILE bildet unter der Bausteingruppe für elementare asynchrone Dateioperationen eine Ausnahme. Im Gegensatz zu den meisten anderen Funktionsbausteinen dieser Gruppe, die eine Dateioperation ausführen, darf eine Instanz von FA_ASYNC_DELETEFILE nicht innerhalb eines Zugriffzyklus aufgerufen werden, da nur das Löschen von Dateien erlaubt ist, auf die aktuell kein Zugriff durch einen anderen Prozeß erfolgt. Die an FA_ASYNC_DELETEFILE übergebene Datei darf deshalb nicht über FA_ASYNC_OPENFILE oder von einem Fremdprogramm aktuell geöffnet sein.
