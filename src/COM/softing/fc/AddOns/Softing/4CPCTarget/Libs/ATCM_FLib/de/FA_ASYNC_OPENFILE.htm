FA_ASYNC_OPENFILE

Öffnet eine Datei zur Bearbeitung.

Schnittstelle

FUNCTION_BLOCK FA_ASYNC_OPENFILE
VAR_INPUT
    EXEC        : BOOL;             
END_VAR
VAR_IN_OUT
    FILE        : FA_ASYNC_FILE;    
END_VAR
VAR_OUTPUT
    DONE        : BOOL;             
    ERROR       : BOOL;
    ERRCD       : DINT;
    ERRTXT      : WSTRING;
END_VAR

Parameter

INPUT

EXEC
Wert vom Typ BOOL
Startet die Operation bei steigender Flanke.

IN_OUT

FILE
Instanz der Struktur FA_ASYNC_FILE
Dateiobjekt, auf das die Operation ausgeführt wird.

OUTPUT

DONE
Wert vom Typ BOOL.
Signalisiert den (erfolgreichen oder gescheiterten) Abschluß der Operation.
Mögliche Werte:
TRUE: Operation ist abgeschlossen.
FALSE: Operation ist noch nicht abgeschlossen.
ERROR
Wert vom Typ BOOL.
Signalisiert den Fehlerstatus.
Mögliche Werte:
TRUE: Ein Fehler ist aufgetreten.
FALSE: Es ist kein Fehler aufgetreten.
ERRCD
Wert vom Typ DINT.
Enthält eine Fehlernummer, falls ein Fehler aufgetreten ist.
Ist kein Fehler aufgetreten, hat ERRCD den Wert 0.
Mögliche Werte sind in den Fehlerkonstanten definiert.
ERRTXT
Wert vom Typ WSTRING.
Enthält einen Fehlertext, falls ein Fehler aufgetreten ist.
Ist kein Fehler aufgetreten, enthält ERRTXT einen Leerstring.

Beschreibung

FA_ASYNC_OPENFILE öffnet die an FILE übergebene Datei für Nur-Lesezugriff oder Lese- und Schreibzugriff. FA_ASYNC_OPENFILE leitet einen Zugriffzyklus ein und muß daher immer aufgerufen werden, bevor Lese- oder Schreibzugriffe über andere Funktionsbausteininstanzen der Bausteingruppe für elementare asynchrone Dateioperationen erfolgen können.

Die zu öffnende Datei wird durch eine Instanz der Struktur FA_ASYNC_FILE repräsentiert. Diese Struktur enthält das Element MODE, über das festgelegt wird, mit welchem Zugriffsmodus eine Datei geöffnet wird. Mögliche Werte sind in den Zugriffsmodus-Konstanten definiert.

Vor dem Öffnen einer Datei muß der Zugriffsmodus in der an FA_ASYNC_OPENFILE übergebenen Strukturinstanz von FA_ASYNC_FILE gesetzt werden. Der beim Öffnen gesetzte Zugriffsmodus ist solange gültig, bis die Datei mit FA_ASYNC_CLOSEFILE wieder geschlossen wird. Alle zwischen dem Öffnen und Schließen einer Datei aufgerufenen Funktionsbausteininstanzen, die auf das an FA_ASYNC_OPENFILE übergebene Dateiobjekt arbeiten, unterliegen den beim Öffnen der Datei in MODE gesetzten Zugriffsberechtigungen. Es ist daher nicht möglich, während eines Zugriffszyklus den Zugriffsmodus zu verändern.

Die Position des Dateizeigers ist beim Öffnen einer Datei immer abhängig von dem in der übergebenen Strukturinstanz von FA_ASYNC_FILE gesetzten Zugriffsmodus. Eine Initialisierung der Elementvariablen POSITION der Struktur FA_ASYNC_FILE mit einem benutzerdefinierten Wert wird beim Öffnen mit FA_ASYNC_OPENFILE durch den dem entsprechenden Zugriffsmodus zugeordneten Wert überschrieben. Sollen Dateizugriffe nach dem Öffnen einer Datei an einer bestimmten Position erfolgen, muß daher nach dem Öffnen der Wert von POSITION explizit neu gesetzt werden.
