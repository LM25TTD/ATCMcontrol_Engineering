FA_ASYNC_RENAMEFILE

Benennt eine Datei um.

Schnittstelle

FUNCTION_BLOCK FA_ASYNC_RENAMEFILE
VAR_INPUT
    EXEC        : BOOL;             
    NEWNAME     : WSTRING;          
END_VAR
VAR_IN_OUT
    FILE        : FA_ASYNC_FILE;    
END_VAR
VAR_OUTPUT
    DONE        : BOOL;             
    ERROR       : BOOL;
    ERRCD       : DINT;
    ERRTXT      : WSTRING;
END_VAR

Parameter

INPUT

EXEC
Wert vom Typ BOOL
Startet die Operation bei steigender Flanke.
NEWNAME
Wert vom Typ WSTRING
Dateiname, in den die Datei umbenannt werden soll.
Der Dateiname kann mit voll qualifiziertem Pfad angegeben werden. Wird kein Pfad spezifiziert, wird intern das Standard-Arbeitsverzeichnis von ATCMControl als Pfad ergänzt.
Wird ein anderer Pfad als der im Parameter FILENAME der an FILE übergebenen Strukturinstanz von FA_ASYNC_FILE spezifizierte Pfad angegeben, wird die umbenannte Datei in das in NEWNAME angegebene Verzeichnis verschoben.
Existiert die in NEWNAME spezifizierte Datei bereits, wird die Operation abgebrochen und ein Fehler gesetzt.
Der Dateiname einschließlich Pfadangabe wird vom Compiler nicht auf korrekte Syntax oder Plausibilität überprüft.

IN_OUT

FILE
Instanz der Struktur FA_ASYNC_FILE
Dateiobjekt, auf das die Operation ausgeführt wird.

OUTPUT

DONE
Wert vom Typ BOOL
Signalisiert den (erfolgreichen oder gescheiterten) Abschluß der Operation.
Mögliche Werte:
TRUE: Operation ist abgeschlossen.
FALSE: Operation ist noch nicht abgeschlossen.
ERROR
Wert vom Typ BOOL
Signalisiert den Fehlerstatus.
Mögliche Werte:
TRUE: Ein Fehler ist aufgetreten.
FALSE: Es ist kein Fehler aufgetreten.
ERRCD
Wert vom Typ DINT
Enthält eine Fehlernummer, falls ein Fehler aufgetreten ist.
Ist kein Fehler aufgetreten, hat ERRCD den Wert 0.
Mögliche Werte sind in den Fehlerkonstanten definiert.
ERRTXT
Wert vom Typ WSTRING
Enthält einen Fehlertext, falls ein Fehler aufgetreten ist.
Ist kein Fehler aufgetreten, enthält ERRTXT einen Leerstring.

Beschreibung

FA_ASYNC_RENAMEFILE benennt eine Datei in den im Parameter NEWNAME spezifizierten Dateinamen um. Die Datei, die umbenannt werden soll, wird im Parameter FILENAME der an FA_ASYNC_RENAMEFILE übergebenen Instanz von FA_ASYNC_FILE angegeben. Nachdem die Datei umbenannt wurde, enthält dieser Parameter den neuen Dateinamen.

FA_ASYNC_RENAMEFILE bildet unter der Bausteingruppe für elementare asynchrone Dateioperationen eine Ausnahme. Im Gegensatz zu den meisten anderen Funktionsbausteinen dieser Gruppe darf eine Instanz von FA_ASYNC_RENAMEFILE nicht innerhalb eines Zugriffzyklus aufgerufen werden, da nur das Umbenennen von Dateien erlaubt ist, auf die aktuell kein Zugriff durch einen anderen Prozeß erfolgt. Die an FA_ASYNC_RENAMEFILE übergebene Datei darf deshalb nicht über FA_ASYNC_OPENFILE oder von einem Fremdprogramm aktuell geöffnet sein.
