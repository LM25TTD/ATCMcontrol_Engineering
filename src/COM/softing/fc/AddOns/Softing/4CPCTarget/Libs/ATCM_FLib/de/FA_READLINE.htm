FA_READLINE

Liest eine Zeile aus einer Datei.

Schnittstelle

FUNCTION_BLOCK FA_READLINE
VAR_INPUT
    EXEC        : BOOL;         
    FILENAME    : WSTRING;      
    TOUT        : TIME;         
END_VAR
VAR_IN_OUT
    POSITION    : DINT;         
END_VAR
VAR_OUTPUT
    DATA        : WSTRING;      
    DONE        : BOOL;         
    ERROR       : BOOL;
    ERRCD       : DINT;
    ERRTXT      : WSTRING;
END_VAR

Parameter

INPUT

EXEC
Wert vom Typ BOOL
Startet die Operation bei steigender Flanke.
FILENAME
Wert vom Typ WSTRING
Spezifiziert den Namen der zu lesenden Datei. Der Dateiname kann einschließlich des voll qualifizierten Pfads angegeben werden. Wird kein Pfad spezifiziert, wird intern das Standard-Arbeitsverzeichnis von ATCMControl als Pfad ergänzt. Das Standard-Arbeitsverzeichnis ist das Verzeichnis data direkt unterhalb des ATCMControl-Installationsverzeichnisses.
TOUT
Wert vom Typ TIME
Definiert die Zeitdauer, während der die Ausführung des Lesevorgangs versucht wird. Nach Ablauf dieses Zeitraums, wird der Versuch, aus der Datei zu lesen, abgebrochen.
Standardwert ist T#0ms. Bei einer Initialisierung mit diesem Wert wird kein Timeout gesetzt, d.h. der Zeitraum, während dessen versucht wird, eine Dateioperation auszuführen, wird nicht überwacht.

IN_OUT

POSITION
Wert vom Typ DINT
Startposition für den Lesezugriff. Nach Beendigung der Leseoperation wird POSITION eine Byte hinter den zuletzt gelesenen Wert gesetzt. Die erste gültige Leseposition ist 0.
Für Beginn und Ende einer Datei sowie das Ende einer Zeile sind Positions-Konstanten definiert.

OUTPUT

DATA
Wert vom Typ WSTRING
Enthält nach Abschluß der Leseoperation die gelesene Zeile.
DONE
Wert vom Typ BOOL
Signalisiert den (erfolgreichen oder gescheiterten) Abschluß der Operation.
Mögliche Werte:
TRUE: Operation ist abgeschlossen.
FALSE: Operation ist noch nicht abgeschlossen.
ERROR
Wert vom Typ BOOL
Signalisiert den Fehlerstatus.
Mögliche Werte:
TRUE: Ein Fehler ist aufgetreten.
FALSE: Es ist kein Fehler aufgetreten.
ERRCD
Wert vom Typ DINT
Enthält eine Fehlernummer, falls ein Fehler aufgetreten ist.
Ist kein Fehler aufgetreten, hat ERRCD den Wert 0.
Mögliche Werte sind in den Fehlerkonstanten definiert.
ERRTXT
Wert vom Typ WSTRING
Enthält einen Fehlertext, falls ein Fehler aufgetreten ist.
Ist kein Fehler aufgetreten, enthält ERRTXT einen Leerstring.

Beschreibung

FA_READLINE liest eine Zeile aus der an FILENAME übergebenen Datei ab der in POSITION spezifizierten Position. Die gelesene Zeile wird in DATA als WSTRING abgelegt. Während des Lesezugriffs wird ein kompletter Zugriffszyklus durchlaufen.

Die Startposition des Lesezugriffs wird in POSITION festgelegt. POSITION repräsentiert den Dateizeiger, der die aktuelle Leseposition markiert.

Nach dem Abschluß einer Leseoperation wird POSITION automatisch auf ein Byte hinter der zuletzt gelesenen Zeile gesetzt. Es ist auf diese Weise möglich, Zeile für Zeile mehrere sequentielle Lesezugriffe durchzuführen, ohne POSITION jedesmal neu zu initialisieren. Dazu wird POSITION beim ersten Aufruf von FA_READLINE eine Variable zugewiesen, die bei jedem folgenden Aufruf von FA_READLINE wieder an deren Instanzvariable POSITION zugewiesen wird. Die Endposition des vorausgehenden Lesezugriffs wird dadurch zur Startposition des folgenden Lesezugriffs. Befindet sich der Dateizeiger am Ende der Datei, hat POSITION den Wert -1 bzw. CFA_EOF.

Die zu lesende Datei wird im Textmodus geöffnet.
