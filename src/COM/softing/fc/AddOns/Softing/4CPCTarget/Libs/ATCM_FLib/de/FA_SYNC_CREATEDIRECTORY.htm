FA_SYNC_CREATEDIRECTORY

Erzeugt ein neues Verzeichnis.

Schnittstelle

FUNCTION FA_SYNC_CREATEDIRECTORY:BOOL
VAR_INPUT
    EXEC        : BOOL;             
END_VAR
VAR_IN_OUT
    FILE        : FA_SYNC_FILE;     
END_VAR

Rückgabewert

Wert vom Typ BOOL
Signalisiert den Fehlerstatus.
Mögliche Werte:
TRUE: Ein Fehler ist aufgetreten.
FALSE: Es ist kein Fehler aufgetreten.

Parameter

INPUT

EXEC
Wert vom Typ BOOL
Startet die Operation.

IN_OUT

FILE
Instanz der Struktur FA_SYNC_FILE
Im Gegensatz zu den anderen Funktionen der Bausteingruppe für elementare synchrone Dateioperationen repräsentiert die an FILE übergebene Strukturinstanz nicht ein Dateiobjekt, auf das die betreffenden Dateioperation ausgeführt wird. Stattdessen wird das Element FILENAME der Struktur FA_SYNC_FILE für die Definition des Verzeichnisnamens genutzt.

Beschreibung

Mit FA_SYNC_CREATEDIRECTORY wird ein neues Verzeichnis angelegt. Eine neue Datei kann mit der Funktion FA_SYNC_OPENFILE nur in einem bereits existierenden Verzeichnis generiert werden. Verwenden Sie die Funktion FA_SYNC_CREATEDIRECTORY, wenn Sie eine neue Datei in einem noch nicht existierenden Verzeichnis generieren wollen, um dieses Verzeichnis zu erzeugen.

Der Name des neu anzulegenden Verzeichnisses wird FA_SYNC_CREATEDIRECTORY über den Parameter FILE übergeben. FILE erwartet eine Instanz der Struktur FA_SYNC_FILE. Diese Struktur enthält das Strukturelement FILENAME, das einen Verzeichnis- oder Dateinamen spezifiziert und vor Übergabe der Strukturinstanz an FA_SYNC_CREATEDIRECTORY initialisiert werden muß.

Bei der Initialisierung des Strukturelements FILENAME für das Erzeugen eines neuen Verzeichnisses darf nur das Verzeichnis ohne Dateinamen angegeben werden. Der Pfad, in dem das Verzeichnis angelegt werden soll, muß voll qualifiziert sein. Die korrekte Syntax von Pfad- und Verzeichnisangabe wird vom Compiler nicht überprüft. Existiert das Verzeichnis bereits, oder ist die Pfad- oder Verzeichnisangabe nicht korrekt, wird zur Laufzeit ein Fehler ausgelöst.

Der Aufruf von FA_SYNC_CREATEDIRECTORY ist unabhängig von einem Zugriffszyklus, d.h. es muß kein vorhergehender Aufruf von FA_SYNC_OPENFILE erfolgen.
