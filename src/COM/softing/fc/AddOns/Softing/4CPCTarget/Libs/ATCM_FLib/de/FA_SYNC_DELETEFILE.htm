FA_SYNC_DELETEFILE

Löscht eine Datei oder ein Verzeichnis.

Schnittstelle

FUNCTION FA_SYNC_DELETEFILE:BOOL
VAR_INPUT
    EXEC        : BOOL;             
END_VAR
VAR_IN_OUT
    FILE        : FA_SYNC_FILE;     
END_VAR

Rückgabewert

Wert vom Typ BOOL
Signalisiert den Fehlerstatus.
Mögliche Werte:
TRUE: Ein Fehler ist aufgetreten.
FALSE: Es ist kein Fehler aufgetreten.

Parameter

INPUT

EXEC
Wert vom Typ BOOL
Startet die Operation.
Mögliche Werte:
TRUE: Operation wird gestartet.
FALSE: Operation wird nicht gestartet.

IN_OUT

FILE
Instanz der Struktur FA_SYNC_FILE
Dateiobjekt, auf das die Operation ausgeführt wird. Die zu löschende Datei wird im Parameter FILENAME der übergebenen Instanz von FA_SYNC_FILE spezifiziert.
Statt einer Datei kann auch ein Verzeichnis gelöscht werden. Zu diesem Zweck wird im Parameter FILENAME der übergebenen Instanz von FA_SYNC_FILE nur ein Verzeichnispfad ohne Dateinamen spezifiziert. Es können nur leere und nicht schreibgeschützte Verzeichnisse gelöscht werden. Ist das spezifizierte Verzeichnis nicht leer oder schreibgeschützt, wird ein Fehler generiert.

Beschreibung

Löscht die an FILE übergebene Datei. Die zu löschende Datei wird im Parameter FILENAME der an FILE übergebenen Strukturinstanz von FA_SYNC_FILE spezifiziert.

FA_SYNC_DELETEFILE bildet unter der Bausteingruppe für elementare synchrone Dateioperationen eine Ausnahme. Im Gegensatz zu den meisten anderen Funktionen dieser Gruppe darf eine Instanz von FA_SYNC_DELETEFILE nicht innerhalb eines Zugriffzyklus aufgerufen werden, da nur das Löschen von Dateien erlaubt ist, auf die aktuell kein Zugriff durch einen anderen Prozeß erfolgt. Die an FA_SYNC_DELETEFILE übergebene Datei darf deshalb nicht über FA_SYNC_OPENFILE oder von einem Fremdprogramm aktuell geöffnet sein.
