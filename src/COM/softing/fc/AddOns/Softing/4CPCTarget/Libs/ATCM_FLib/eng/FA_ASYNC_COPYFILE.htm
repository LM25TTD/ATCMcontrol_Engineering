FA_ASYNC_COPYFILE

Copies a file to the specified target directory.

Interface

FUNCTION_BLOCK FA_ASYNC_COPYFILE

VAR_INPUT
    EXEC        : BOOL;
    DESTNAME    : WSTRING;
    OVERWRITE   : BOOL := TRUE;     
END_VAR
VAR_IN_OUT
    FILE        : FA_ASYNC_FILE;    
END_VAR
VAR_OUTPUT
    DONE        : BOOL;
    ERROR       : BOOL;
    ERRCD       : DINT;
    ERRTXT      : WSTRING;
END_VAR

Parameter

INPUT

EXEC
Value of the type BOOL
Starts the operation at a rising edge.
DESTNAME
Value of the type WSTRING
Specifies the target directory and the name that the copied file is to receive. If no target directory is specified, the file is copied under the specified name to the standard working directory of ATCMControl. This directory has the name data and is located directly under the ATCMControl installation directory.
OVERWRITE
Value of the type BOOL
Defines whether a file is to be overwritten if it already exists.
Possible values:
TRUE: An existing file is overwritten.
FALSE: An existing file is not overwritten. The process is aborted if a file of the same name already exists in the target directory.
The default value is TRUE.

IN_OUT

FILE
Instance of the FA_ASYNC_FILE structure
Specifies the file to be copied.

OUTPUT

DONE
Value of the type BOOL
Signalizes the (successful or failed) conclusion of the operation.
Possible values:
TRUE: Operation has been concluded.
FALSE: Operation has not yet been concluded.
ERROR
Value of the type BOOL
Signalizes the error status.
Possible values:
TRUE: An error has occurred.
FALSE: No error has occurred.
ERRCD
Value of the type DINT
Contains an error number if an error has occurred.
If no error has occurred, ERRCD has the value 0.
The possible values are defined in the error constants.
ERRTXT
Value of the type WSTRING
Contains an error text if an error has occurred.
If no error has occurred, ERRTXT contains an empty string.

Description

FA_ASYNC_COPYFILE copies the file given in the FILE parameter to the target directory specified in DESTNAME and renames the file, if applicable. The source file does not need to be opened before copying, which means that the execution of FA_ASYNC_COPYFILE is independent of an access cycle.
