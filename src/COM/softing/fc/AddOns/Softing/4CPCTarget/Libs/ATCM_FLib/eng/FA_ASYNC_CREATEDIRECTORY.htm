FA_ASYNC_CREATEDIRECTORY

Creates a new directory.

Interface

FUNCTION_BLOCK FA_ASYNC_CREATEDIRECTORY
VAR_INPUT
    EXEC        : BOOL;
END_VAR
VAR_IN_OUT
    FILE        : FA_ASYNC_FILE;    
END_VAR
VAR_OUTPUT
    DONE        : BOOL;
    ERROR       : BOOL;
    ERRCD       : DINT;
    ERRTXT      : WSTRING;
END_VAR

Parameter

INPUT

EXEC
Value of the type BOOL
Starts the operation at a rising edge

IN_OUT

FILE
Instance of the FA_ASYNC_FILE structure
In contrast to the other functions provided in the group of blocks for elementary asynchronous file operations, the structure instance passed to FILE does not represent a file object on which the corresponding file operation is performed. The FILENAME element of the FA_ASYNC_FILE structure is instead used for defining the directory name.

OUTPUT

DONE
Value of the type BOOL
Signalizes the (successful or failed) conclusion of the operation.
Possible values:
TRUE: Operation has been concluded.
FALSE: Operation has not yet been concluded.
ERROR
Value of the type BOOL
Signalizes the error status.
Possible values:
TRUE: An error has occurred.
FALSE: No error has occurred.
ERRCD
Value of the type DINT
Contains an error number if an error has occurred.
If no error has occurred, ERRCD has the value 0.
The possible values are defined in the error constants.
ERRTXT
Value of the type WSTRING
Contains an error text if an error has occurred.
If no error has occurred, ERRTXT contains an empty string.

Description

With FA_ASYNC_CREATEDIRECTORY you create a new directory. A new file can only be generated with an instance of the function block FA_ASYNC_OPENFILE if the directory in which it is to be generated already exists. If you want to generate a new file in a directory that does not yet exist, use an instance of FA_ASYNC_CREATEDIRECTORY to create this directory.

The name of the directory to be newly created is passed to FA_ASYNC_CREATEDIRECTORY in the parameter FILE. FILE expects an instance of the FA_ASYNC_FILE structure. This structure contains the structure element FILENAME which specifies a directory or file name and must be initialized before the structure instance is passed to FA_ASYNC_CREATEDIRECTORY.

When you initialize the structure element FILENAME for creating a new directory, you must enter only the directory, without the file name. The path in which the directory is to be created must be fully qualified (e.g. C:\Projects). The compiler does not check whether the syntax of the path and directory information is correct. If the directory already exists or if the path or directory information is not correct, an error will be output during runtime.
