FA_ASYNC_DELETEFILE

Deletes a file or directory.

Interface

FUNCTION_BLOCK FA_ASYNC_DELETEFILE
VAR_INPUT
    EXEC        : BOOL;
END_VAR
VAR_IN_OUT
    FILE        : FA_ASYNC_FILE;    
END_VAR
VAR_OUTPUT
    DONE        : BOOL;
    ERROR       : BOOL;
    ERRCD       : DINT;
    ERRTXT      : WSTRING;
END_VAR

Parameter

INPUT

EXEC
Value of the type BOOL
Starts the operation at a rising edge.

IN_OUT

FILE
Instance of the FA_ASYNC_FILE structure
File object on which the operation is performed. The file to be deleted is specified in the parameter FILENAME of the passed instance of FA_ASYNC_FILE.
Instead of a file you can also delete a directory. This is done by specifying a directory path only, without a file name, in the parameter FILENAME of the passed instance of FA_ASYNC_FILE; e.g., “D:\Projects\Testproject”. You can delete only directories that are empty and not write-protected. If the specified directory is not empty or if it is write-protected, an error is generated.

OUTPUT

DONE
Value of the type BOOL
Signalizes the (successful or failed) conclusion of the operation.
Possible values:
TRUE: Operation has been concluded.
FALSE: Operation has not yet been concluded.
ERROR
Value of the type BOOL
Signalizes the error status.
Possible values:
TRUE: An error has occurred.
FALSE: No error has occurred.
ERRCD
Value of the type DINT
Contains an error number if an error has occurred.
If no error has occurred, ERRCD has the value 0.
The possible values are defined in the error constants.
ERRTXT
Value of the type WSTRING
Contains an error text if an error has occurred.
If no error has occurred, ERRTXT contains an empty string.

Description

Deletes the file passed to FILE. The file to be deleted is specified in the parameter FILENAME of the structure instance of FA_ASYNC_FILE which is passed to FILE.

FA_ASYNC_DELETEFILE constitutes an exception among the group of blocks for elementary asynchronous file operations. In contrast to most other function blocks of this group which perform a file operation, an instance of FA_ASYNC_DELETEFILE must not be called within an access cycle since only files that are currently not accessed by another process may be deleted. The file passed to FA_ASYNC_DELETEFILE therefore must not be currently open due to FA_ASYNC_OPENFILE or due to an external program.
