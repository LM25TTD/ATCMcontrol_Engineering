FA_ASYNC_GETSIZE

Delivers the size of a file in bytes.

Interface

FUNCTION_BLOCK FA_ASYNC_GETSIZE
VAR_INPUT
    EXEC        : BOOL;
END_VAR
VAR_IN_OUT
    FILE        : FA_ASYNC_FILE;    
END_VAR
VAR_OUTPUT
    SIZE        : LINT;
    DONE        : BOOL;
    ERROR       : BOOL;
    ERRCD       : DINT;
    ERRTXT      : WSTRING;
END_VAR

Parameter

INPUT

EXEC
Value of the type BOOL
Starts the operation at a rising edge.

IN_OUT

FILE
Instance of the FA_ASYNC_FILE structure.
File object on which the operation is performed.

OUTPUT

SIZE
Value of the type LINT
Contains the size in bytes of the file passed to FILE on conclusion of the operation.
DONE
Value of the type BOOL
Signalizes the (successful or failed) conclusion of the operation.
Possible values:
TRUE: Operation has been concluded.
FALSE: Operation has not yet been concluded.
ERROR
Value of the type BOOL
Signalizes the error status.
Possible values:
TRUE: An error has occurred.
FALSE: No error has occurred.
ERRCD
Value of the type DINT
Contains an error number if an error has occurred.
If no error has occurred, ERRCD has the value 0.
The possible values are defined in the error constants.
ERRTXT
Value of the type WSTRING
Contains an error text if an error has occurred.
If no error has occurred, ERRTXT contains an empty string.

Description

In the parameter SIZE, FA_ASYNC_GETSIZE delivers the size of the file passed to FILE. The file does not need to be opened before determining the file size, which means that the execution of FA_ASYNC_GETSIZE is independent of an access cycle.
