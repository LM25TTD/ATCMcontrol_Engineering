FA_ASYNC_RENAMEFILE

Renames a file.

Interface

FUNCTION_BLOCK FA_ASYNC_RENAMEFILE
VAR_INPUT
    EXEC        : BOOL;
    NEWNAME     : WSTRING;
END_VAR
VAR_IN_OUT
    FILE        : FA_ASYNC_FILE;    
END_VAR
VAR_OUTPUT
    DONE        : BOOL;
    ERROR       : BOOL;
    ERRCD       : DINT;
    ERRTXT      : WSTRING;
END_VAR

Parameter

INPUT

EXEC
Value of the type BOOL
Starts the operation at a rising edge.
NEWNAME
Value of the type WSTRING
Name to which the file is to be renamed.
The file name can be indicated by a fully qualified path. If no path is specified, the standard working directory of ATCMControl is internally added as the path.
If the entered path differs from the path specified in the parameter FILENAME of the structure instance of FA_ASYNC_FILE which is passed to FILE, the renamed file is moved to the directory indicated in NEWNAME.
If the file specified in NEWNAME already exists, the operation is aborted and an error is output.
The compiler does not check whether the syntax of the file name and path information is correct or whether they are plausible.

IN_OUT

FILE
Instance of the FA_ASYNC_FILE structure
File object on which the operation is performed.

OUTPUT

DONE
Value of the type BOOL
Signalizes the (successful or failed) conclusion of the operation.
Possible values:
TRUE: Operation has been concluded.
FALSE: Operation has not yet been concluded.
ERROR
Value of the type BOOL
Signalizes the error status.
Possible values:
TRUE: An error has occurred.
FALSE: No error has occurred.
ERRCD
Value of the type DINT
Contains an error number if an error has occurred.
If no error has occurred, ERRCD has the value 0.
The possible values are defined in the error constants.
ERRTXT
Value of the type WSTRING
Contains an error text if an error has occurred.
If no error has occurred, ERRTXT contains an empty string.

Description

FA_ASYNC_RENAMEFILE renames a file the name specified in the parameter NEWNAME. The file to be renamed is given in the parameter FILENAME of the instance of FA_ASYNC_FILE passed to FA_ASYNC_RENAMEFILE. After the file has been renamed, this parameter receives the new file name.

FA_ASYNC_RENAMEFILE constitutes an exception among the group of blocks for elementary asynchronous file operations. In contrast to most other function blocks of this group, an instance of FA_ASYNC_RENAMEFILE must not be called within an access cycle since only files that are currently not accessed by another process may be renamed. The file passed to FA_ASYNC_RENAMEFILE therefore must not be currently open due to FA_ASYNC_OPENFILE or due to an external program.
