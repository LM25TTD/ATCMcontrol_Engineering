FA_SYNC_CREATEDIRECTORY

Creates a new directory.

Interface

FUNCTION FA_SYNC_CREATEDIRECTORY:BOOL
VAR_INPUT
    EXEC        : BOOL;
END_VAR
VAR_IN_OUT
    FILE        : FA_SYNC_FILE;
END_VAR

Return value

Value of the type BOOL
Signalizes the error status.
Possible values:
TRUE: An error has occurred.
FALSE: No error has occurred.

Parameter

INPUT

EXEC
Value of the type BOOL
Starts the operation.

IN_OUT

FILE
Instance of the FA_SYNC_FILE structure
In contrast to the other functions provided in the group of blocks for elementary synchronous file operations, the structure instance passed to FILE does not represent a file object on which the corresponding file operation is performed. The FILENAME element of the FA_SYNC_FILE structure is instead used for defining the directory name.

Description

With FA_SYNC_CREATEDIRECTORY you create a new directory. A new file can only be generated with the function FA_SYNC_OPENFILE if the directory in which it is to be generated already exists. If you want to generate a new file in a directory that does not yet exist, use the function FA_SYNC_CREATEDIRECTORY to create this directory.

The name of the directory to be newly created is passed to FA_SYNC_CREATEDIRECTORY in the parameter FILE. FILE expects an instance of the FA_SYNC_FILE structure. This structure includes the structure element FILENAME which specifies a directory or file name and must be initialized before the structure instance is passed to FA_SYNC_CREATEDIRECTORY.

When you initialize the structure element FILENAME for creating a new directory, you must enter only the directory, without the file name. The path in which the directory is to be created must be fully qualified. The compiler does not check whether the syntax of the path and directory information is correct. If the directory already exists or if the path or directory information is not correct, an error will be output during runtime.

The call of FA_SYNC_CREATEDIRECTORY is independent of an access cycle, which means that a previous call of FA_SYNC_OPENFILE is not required.
