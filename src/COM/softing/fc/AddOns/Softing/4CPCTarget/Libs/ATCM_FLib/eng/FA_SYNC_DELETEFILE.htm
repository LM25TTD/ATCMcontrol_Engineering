FA_SYNC_DELETEFILE

Deletes a file or directory.

Interface

FUNCTION FA_SYNC_DELETEFILE:BOOL
VAR_INPUT
    EXEC        : BOOL;
END_VAR
VAR_IN_OUT
    FILE        : FA_SYNC_FILE;
END_VAR

Return value

Value of the type BOOL
Signalizes the error status.
Possible values:
TRUE: An error has occurred.
FALSE: No error has occurred.

Parameter

INPUT

EXEC
Value of the type BOOL
Starts the operation.
Possible values:
TRUE: Operation is started.
FALSE: Operation is not started.

IN_OUT

FILE
Instance of the FA_SYNC_FILE structure
File object on which the operation is performed. The file to be deleted is specified in the parameter FILENAME of the passed instance of FA_SYNC_FILE.
Instead of a file you can also delete a directory. This is done by specifying a directory path only, without a file name, in the parameter FILENAME of the passed instance of FA_SYNC_FILE; e.g., “D:\Projects\Testproject”. You can delete only directories that are empty and not write-protected. If the specified directory is not empty or if it is write-protected, an error is generated.

Description

Deletes the file passed to FILE. The file to be deleted is specified in the parameter FILENAME of the structure instance of FA_SYNC_FILE which is passed to FILE.

FA_SYNC_DELETEFILE constitutes an exception among the group of blocks for elementary synchronous file operations. In contrast to most other functions of this group, an instance of FA_SYNC_DELETEFILE must not be called within an access cycle since only files that are currently not accessed by another process may be deleted. The file passed to FA_SYNC_DELETEFILE therefore must not be currently open due to FA_SYNC_OPENFILE or due to an external program.
