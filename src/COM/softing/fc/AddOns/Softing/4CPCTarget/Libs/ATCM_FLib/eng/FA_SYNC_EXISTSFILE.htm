FA_SYNC_EXISTSFILE

Checks whether a file exists.

Interface

FUNCTION FA_SYNC_EXISTSFILE:BOOL
VAR_INPUT
    EXEC        : BOOL;
END_VAR
VAR_IN_OUT
    FILE        : FA_SYNC_FILE;
END_VAR
VAR_OUTPUT
    EXISTS      : BOOL;
END_VAR

Return value

Value of the type BOOL
Signalizes the error status.
Possible values:
TRUE: An error has occurred.
FALSE: No error has occurred.

Parameter

INPUT

EXEC
Value of the type BOOL
Starts the operation.

IN_OUT

FILE
Instance of the FA_SYNC_FILE structure
In contrast to most of the other functions provided in the group of blocks for elementary synchronous file operations, the structure instance passed to FILE does not represent a file object on which the corresponding file operation is performed. The FILENAME element of the FA_SYNC_FILE structure is used instead for defining the name of the file to be found.

OUTPUT

EXISTS
Value of the type BOOL
Indicates, after executing the operation, whether the file specified in the element FILENAME of the structure instance of FA_SYNC_FILE passed to FILE exists.
Possible values:
TRUE: The file exists.
FALSE: The file does not exist.

Description

FA_SYNC_EXISTSFILE checks whether a file exists.

The name of the file to be found is passed to FA_SYNC_EXISTSFILE in the parameter FILE. FILE expects an instance of the FA_SYNC_FILE structure. This structure includes the structure element FILENAME which specifies a file name and must be initialized before the structure instance is passed to FA_SYNC_EXISTSFILE .

The call of FA_SYNC_EXISTSFILE is independent of an access cycle, which means that a previous call of FA_SYNC_OPENFILE is not required.
