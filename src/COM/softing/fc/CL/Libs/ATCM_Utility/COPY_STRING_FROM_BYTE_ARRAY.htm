COPY_STRING_FROM_BYTE_ARRAY

Reads a defined number of characters, starting from a specific position in a byte array, and converts and copies the read bytes into a STRING.

Interface

FUNCTION COPY_STRING_FROM_BYTE_ARRAY : STRING
VAR_INPUT
    NUM : DINT;                                 
END_VAR
VAR_IN_OUT
    POS : DINT;                                 
    BYTES : ARRAY OF BYTE;
END_VAR

Return value

Value of the type STRING
Value that was read from the byte array.

Parameter

INPUT

NUM
Value of the type DINT
Specifies the number of characters to be delivered in the return value.

IN_OUT

POS
Value of the type DINT
Specifies as from which position in the byte array the value is to be read. Counting starts from 0 for the first position in the byte array and is performed in byte units.
After the successful execution of the function, POS is overwritten by the value of the first byte array position that follows the last byte read. If the execution of the function failed, POS receives the value -1.
BYTES
Array of the type BYTE
Byte array from which a value is to be read.

Description

COPY_STRING_FROM_BYTE_ARRAY reads a value of the type STRING from the byte array passed to BYTES as from the position passed to POS, and delivers this value as the return value.

Starting from the position POS in the byte array, the characters are copied to the string. POS is incremented accordingly during copying.

Please note:

  • The number of characters that are to be read is defined by the maximum size of the string that is returned by the function, i.e. by the size which was specified in its definition or declaration.

  • The definition str : STRING is equivalent to str : STRING[MAX_STRING_LEN], where MAX_STRING_LEN depends on which target system is being used. The default value for MAX_STRING_LEN is 255.

  • The size of the byte array, the maximum size of the string and the starting position must be matched to one another. If the byte array is too small and the string cannot be completely filled from the byte array, no characters will be copied and the task will be stopped with a fatal error.
