Connection parameters

Parameters defining the properties of a new connection

Syntax

Single parameters are separated by a comma . The parameters are defined by their position in the connection string:
<Port nb>,<Baudrate>,<Data bits>,<Parity>,<Stop bits>,<Protocol>

Example: Connectionstring := "1,9600,8,N,1,NO"

Meaning of the parameters

Parameter Meaning Possible values
<Port nb> Number of the COM-Port 1 - 9
<Baudrate> Baudrate 110, 300, 600, 1200, 2400, 4800, 9600, 14400, 19200, 38400, 56000, 57600, 115200, 128000, 230400, 256000, 460800, 921600
<Data bits> Number of Bits per Date 4, 5, 6, 7, 8
<Parity> Parity N none
E even
O odd
M marker
S space
<Stop bits> Nuumber of Stopbits 1, 1.5, 2
<Protocol> Protocol used NO    no protocol
SW     Xon/Xoff (Software-Handshake)
HW    RTS/CTS (Hardware-Handshake)

If a parameter is missing, the following default values are used:

  • Port nb:1
  • Baudrate: 9600
  • Databits: 8
  • Parity: N
  • Stopbits: 1
  • Protocol: NO  

Description

The connection parameters of the connection string are passed to the parameter sParam of an instance of the structure COMM_PARTNER. They define central properties for a serial connection established with CONNECT.
