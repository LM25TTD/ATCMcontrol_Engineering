Hinweise zu Windows XP SP2 /
Windows 2003 Server SP1

ATCMControl PC Control & Datenausführungsverhinderung (DEP)

Um Eine einwandfreie Funktion des ATCM PC Controls zu gewährleisten, sollte die Boot-Option /noexecute=OptIn im Abschnitt [operating systems] in der Datei „Boot.ini“ gesetzt werden, da es sonst zu Schwierigkeiten beim Start des Control-Dienstes nach dem Bootvorgang kommen kann.

Wenn Sie als Administrator angemeldet sind, können Sie die Boot-Option OptIn manuell auf der Registerkarte Datenausführungsverhinderung im Dialogfeld Systemeigenschaften konfigurieren.
Im folgenden Verfahren wird die manuelle Konfiguration der DEP Einstellungen für ATCMControl PC Control auf dem Computer beschrieben:

  1. Klicken Sie im Startmenü auf Systemsteuerung, und doppelklicken Sie dann auf System.

  2. Klicken Sie auf die Registerkarte Erweitert. Klicken Sie dann unter Leistung auf Einstellungen.

  3. Klicken Sie auf die Registerkarte Datenausführungsverhinderung.

  4. Klicken Sie auf Datenausführungsverhinderung nur für erforderliche Windows-Programme und -Dienste aktivieren, um die Richtlinie OptIn auszuwählen.


     

 

Falls Sie an weiteren Informationen zur Datenausführungsverhinderung und deren Konfiguration interessiert sind können Sie diese in den nachfolgenden Abschnitten nachlesen.
 

DEP (Data Execution Prevention)

In Windows XP SP2 und Windows 2003 Server SP1 wurden von Microsoft einige Sicherheitstechnologien zur Datenausführungsverhinderung (Data Execution Prevention, DEP) implementiert, welche die Ausführung von Programmcode in bestimmten Speicherbereichen verhindern.

Es gibt zwei Arten der DEP. Zu einen die hardwaregesteuerte DEP und zum Anderen die softwaregesteuerte DEP.
In Microsoft Windows XP Servicepack 2 (SP2) und Microsoft Windows 2003 Server Servicepack 1 (SP1) wird die Datenausführungsverhinderung durch Hardware und Software erzwungen.

Hardwarebasierte DEP

Neuere 64-Bit CPU's unterstützen die harwaregesteuerte Datenausführungsverhinderung (DEP). Sie haben die Möglichkeit bestimmte Speicherbereiche als Datenbereiche zu markieren und die Ausführung von Programmcode, aus diesen Bereichen heraus, zu verhindern.
Die hardwaregesteuerte Datenausführungsverhinderung ist auf Computern mit DEP-kompatiblen Prozessoren, die Microsoft Windows XP 64-Bit Edition ausführen, standardmäßig aktiviert.
 

Softwarebasierte DEP

Windows XP SP2 und Windows 2003 SP1 enthalten eine Reihe von zusätzlichen DEP-Sicherheitsprüfungsfunktionen, die einen Mißbrauch von in Windows verankerten Mechanismen zur Ausnahmebehandlung vorbeugen sollen.
Die softwarebasierte DEP wird von jedem Prozessor unterstützt, der Windows Server 2003 mit Service Pack 1 bzw. Windows XP SP2 ausführen kann.
Die durch Software erzwungene Datenausführungsverhinderung schützt unter Windows XP SP2 standardmäßig nur bestimmte Systembinärdateien. Bei Windows Server 2003 SP1 als Serverbetreibssystem werden standardmäßig alle Anwendungen und Dienste geschützt.
Die softwaregesteuerte DEP funktioniert unabhängig von den jeweiligen DEP-Funktionen des Prozessors.
 

Systemweite Konfiguration der DEP

Die Konfiguration der DEP für das System wird über Optionen in der Datei „Boot.ini“ gesteuert.
Endbenutzer können die Einstellungen der DEP leicht über die Eigenschaften des Systems in der Systemsteuerung konfigurieren, wenn sie als Administrator angemeldet sind.
Die Systemkonfigurationseinstellungen der DEP gelten nur für 32-Bit-Anwendungen und -Prozesse, die unter 32-Bit- oder 64-Bit-Versionen von Windows ausgeführt werden. Unter 64-Bit-Versionen von Windows wird die durch Hardware erzwungene DEP, wenn sie zur Verfügung steht, immer auf 64-Bit-Prozesse und Kernelspeicherbereiche angewendet und kann nicht durch Systemkonfigurationseinstellungen deaktiviert werden.
Windows unterstützt vier systemweite Konfigurationen für durch Hardware und Software erzwungene DEP.
 

Konfiguration Beschreibung

OptIn

(Standard für Windows XP SP2 und Windows XP 64-Bit-Edition)

Auf Systemen mit Prozessoren, die die durch Hardware erzwungene DEP unterstützen, wird diese standardmäßig für bestimmte Systembinärdateien und Anwendungen aktiviert, die hierfür konfiguriert wurden.
Mit dieser Option wird die DEP standardmäßig nur auf Windows-Systembinärdateien angewendet.

OptOut

(Standard für Windows Server 2003 Service Pack 1)

Die DEP ist standardmäßig für alle Prozesse aktiviert. Benutzer können über System in der Systemsteuerung manuell eine Liste mit bestimmten Anwendungen erstellen, auf die die DEP nicht angewendet wird. IT-Experten können mithilfe des Application Compatibility Toolkit eine oder mehrere Anwendungen vom Schutz durch die DEP ausnehmen. Systemkompatibilitätskorrekturen (shims) für die DEP werden wirksam.

AlwaysOn

Die DEP gilt für das gesamte System. Die DEP wird beim Ausführen aller Prozesse angewendet. Die Ausnahmenliste für das Ausnehmen bestimmter Anwendungen vom Schutz durch die DEP steht nicht zur Verfügung. Systemkompatibilitätskorrekturen (shims) für die DEP werden nicht wirksam. Anwendungen, die mithilfe des Application Compatibility Toolkit ausgenommen wurden, werden mit angewendeter DEP ausgeführt.

AlwaysOff

Für das gesamte System gilt unabhängig von der DEP-Unterstützung der Hardware kein Schutz durch die DEP. Der Prozessor wird bei 32-Bit-Versionen von Windows im PAE-Modus ausgeführt, es sei denn, die Option /noexecute=alwaysoff wird durch die Option /execute im Starteintrag ersetzt.

 

Folgende Einstellungen gelten für die Datei „Boot.ini“:

/noexecute=policy_level

Dabei entspricht der Parameter policy_level einem der Werte aus der obigen Tabelle.
Die Einstellung /noexecute in der Datei „Boot.ini“ wird, sofern vorhanden, beim Installieren von Windows XP SP2 bzw. Windows 2003 Server SP1 nicht geändert. Die Einstellungen werden auch beim Verschieben eines Windows-Betriebssystemabbilds zwischen Computern mit und ohne Unterstützung für durch Hardware erzwungene DEP nicht geändert.

Wenn Sie als Administrator angemeldet sind, können Sie die DEP zwischen den Richtlinien OptIn und OptOut manuell auf der Registerkarte Datenausführungsverhinderung im Dialogfeld Systemeigenschaften konfigurieren.
Im folgenden Verfahren wird die manuelle Konfiguration der DEP auf dem Computer beschrieben:

  1. Klicken Sie im Startmenü auf Systemsteuerung, und doppelklicken Sie dann auf System.

  2. Klicken Sie auf die Registerkarte Erweitert. Klicken Sie dann unter Leistung auf Einstellungen.

  3. Klicken Sie auf die Registerkarte Datenausführungsverhinderung.

  4. Klicken Sie auf Datenausführungsverhinderung nur für erforderliche Windows-Programme und -Dienste aktivieren, um die Richtlinie OptIn auszuwählen.


     

  5. Klicken Sie auf Datenausführungsverhinderung für alle Programme und Dienste mit Ausnahme der ausgewählten aktivieren, um die Richtlinie OptOut auszuwählen.


     

  6. Wenn Sie die Richtlinie OptOut ausgewählt haben, klicken Sie auf Hinzufügen, und fügen Sie die Anwendungen hinzu, für die die Datenausführungsverhinderung nicht verwendet werden soll.


 

IT-Experten können die systemweite Konfiguration der DEP mit einer Reihe von Methoden steuern. Die Datei „Boot.ini“ kann direkt mit Skriptmechanismen oder mit dem im Windows enthaltenen Tool Bootcfg.exe geändert werden.

Gehen Sie folgendermaßen vor, um zur Konfiguration der DEP mithilfe der Datei „Boot.ini“ zur Richtlinie OptIn zu wechseln.

  1. Klicken Sie auf Start, klicken Sie mit der rechten Maustaste auf Arbeitsplatz, und klicken Sie auf Eigenschaften.

  2. Klicken Sie in der Registerkarte Erweitert unter Starten und Wiederherstellen auf Einstellungen.

  3. Klicken Sie unter Systemstart auf Bearbeiten. Die Datei „Boot.ini“ wird in Microsoft Editor geöffnet.

  4. Ersetzen Sie den policy_level durch OptIn

    WARNUNG Achten Sie darauf, den Text exakt einzugeben. Die Option für die Datei „Boot.ini“ sollte nun folgendermaßen lauten:

    /noexecute=OptIn

  5. Klicken Sie im Editor im Menü Datei auf Speichern.

  6. Klicken Sie zweimal auf OK.

  7. Starten Sie den Computer neu.
