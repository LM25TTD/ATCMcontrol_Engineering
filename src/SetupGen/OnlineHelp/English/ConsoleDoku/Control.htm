Establishing/Interrupting a Connection

Establishing a connection

For initialization of the variables, each ATCM Control must be connected with ATCM Engineering. In ATCM Engineering, ATCM Control is represented as a PC. To be able to monitor the variable and its messages in the Watch view of ATCM Engineering, you have to be connected to the corresponding ATCM Control (PC). In this case, you work online, i.e. with an established connection.

Go to the Message view and click on the respective PC tab.
Then click on the Connect button to establish the connection to the PC.

- OR -

click on Connect and the desired PC in the menu item Control in the main menu.

As soon as the PC is connected with the ATCM Engineering tool, the PC symbol in the Navigator view changes to the PC-Connect symbol.

Interrupting a connection

If you interrupt the connection between the PC and the ATCM Engineering tool, you can no longer monitor its variables and messages in the Watch view of the ATCM Engineering tool. You can also work offline, i.e. without connection to the PC; it is not necessary for you to be online when you edit or compile a project.

Go to the Message view and click on the respective PC tab.
Then click on the Disconnect button to interrupt the connection to the PC

- OR -

click on Disconnect and the desired PC in the menu item Control in the main menu.

Enabling a debug connection

Before you can debug and download your code onto ATCM Control, a Debug connection must be established. During a debug connection, only one ATCM Engineering is permitted to be connected to one ATCM Control. Remember to interrupt the debug connection after having finished a download or a debugging session so that a connection can be established from other ATCM Engineerings as well.

Go to the Message view and click on the respective PC tab.
Click on the Enable Debug button to establish a new debug connection,

- OR -

click on Enable Debug in the Control menu in the menu bar. Then select the desired PC.

When the debug connection has been established, the connection symbol in the Message view changes to the symbol.

Interrupting a debug connection

When you interrupt the debug connection, the exclusive connection to the control is also interrupted. This means that the control continues to be connected to ATCM Engineering, but other ATCM Engineerings now have access to the control as well.

Go to the Message view and click on the respective PC tab.
Click on the Disable Debug button to interrupt the exclusive connection

- OR -

click on Disable Debug in the Control menu in the menu bar and select the desired PC.

Starting/Stopping all Tasks

Before you can use the start/stop function, a connection to the PC must have been established.

For trouble shooting during project installation, it may be necessary to start and stop ATCM Control on a PC. Starting or stopping ATCM Control means to start or stop, respectively, all resources of the assigned configuration simultaneously.
The starting/stopping function should not be used during normal operation but only during the installation of the project.

In the main menu, go to the menu item Control. Depending on the current action of the configuration, click either on Start all Tasks or on Stop all Tasks.
If ATCM Console is opened, you can monitor the status signals of the tasks in the Task list faceplate and in the Resource list. You can display these by clicking on the project symbol in the Navigator of ATCM Console.
Now you have started or stopped, respectively, all tasks of this resource.

Running/Stopping ATCM Control

To stop a running ATCM Control, go to the menu item Control in the main menu and click on Stop Control. ATCM Control is stopped as shown by its symbol .

Stopping ATCM Control offers the following advantages:

  • The connection to all PROFIBUS slaves is interrupted. This gives you the possibility of reconfiguring the slaves or configuring additional ones.
  • You can operate or write all variables in ATCM Console or ATCM Engineering, even when ATCM Control is in a resting phase.
  • All current variable values are stored. You can change the value of each variable without changing the project. This gives you the possibility of trying out different values and observing the results. If the values do not fit, just reset them. As soon as you start ATCM Control, all altered variables are reset to their previous values.
  • The configuration components do not start automatically after the download. You can compile the entire configuration and start after having configured all components.

To start ATCM Control, go to the menu item Control in the main menu and click on Run Control.
The ATCM Control symbol changes from yellow to green.

Reset with Cold Start/Warm Start

Cold start

If the current project does not run properly on ATCM Control, it may be necessary to reset all variables. By means of the cold start function, you can restart the project; this means that all variables are reset to their initial values.

This includes even variables secured against system failure (VAR RETAIN).

In the menu item Control of the main menu, click on Cold Start Reset and the desired PC.
All variables are reset and the project is restarted with the initial values.

Warm start

With the warm start function, you can reset all variables loaded onto ATCM Control to their initial values. The only exception are the retain variables (VAR RETAIN) which are written on the hard disk in regular intervals for reasons of safety. They are initialized with the last valid value before the warm start.

Only the variables secured against system failure (VAR RETAIN) retain their values during restart; all other variables are reset to their initial values during this process.

In the menu item Control of the main menu, click on Warm Start Reset and the desired PC.

Initializing/Deleting

If you have to delete the running project from ATCM Control, use the Clear function to do this. This function deletes the entire ATCM Control. All project components are deleted so that ATCM Control retains no parts of the project.

Go to the Message view, click on the tab of the corresponding PC and then on the Initialize/Clear button.

- OR -

click on Initialize/Clear in the menu item Control of the main menu.
The ATCM Control on the selected PC is deleted.

Context Menu of ATCM Control

The ATCM Context symbol in the task menu of Windows NT includes three different menu items.

Go to the ATCM Control symbol in the task menu and press the right mouse button.
The Context menu is displayed:

To use one of the following menu items, just click on it.

Open ATCM Console 

You can call ATCM Console by means of a mouse click on this menu item.

ATCM Control License 

If you click on this menu item, a message dialog with information about your license is displayed.

You will see all types of information concerning your license.
To be able to use ATCM Control with a full license, you also have to enter the enable code here.
For this purpose, send the product serial no. to Softing GmbH.
Click on Clipboard to copy the product serial no. to the clipboard.
Send this number to ATCM, c/o ATCMControl.
As soon as you have received your enable code from Softing GmbH, click on Enable to enter the code.

Enter the enable code and confirm with OK.

Close ATCM Tray 

Click on this menu item to hide the ATCM Control symbol. This does not stop or close ATCM Control; it only causes the symbol to disappear.

If you want to redisplay the ATCM Control symbol, go to the Windows start menu.
  1. Click on Run... .
  2. Enter the path and the file name "4ctray.exe", e.g.:
    C:/Program/4CONTROL/CTL/4Ctray.exe
  3. Click on OK.
  4. The ATCM Control symbol reappears.
