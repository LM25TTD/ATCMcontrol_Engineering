Working with Layouts

Layouts form the basic structure for the arrangement of the displays shown in the Visualization view. The arrangement of displays can be altered via the layout editor or via the Context menus of the displays in the Visualization view.

Alterations of layouts are always carried out on the basis of the currently enabled layout. The enabled layout is indicated in the Layout list box in the ATCM Control toolbar. If you have not yet stored any new layout, the basic layout is the standard layout used by ATCM Control.

Alterations of the layout which have been carried through via the layout editor or via the Context menu of the displays are not automatically stored. As soon as you update the entire Visualization view via the corresponding option in the Internet Explorer or restart ATCM Console, the displays are again arranged in the order of the currently loaded layout. To maintain layout changes, you have to explicitly store the layout.
For saving and loading a layout, see also the sections: Saving Altered Layouts and Loading a saved Layout.

Every time you start ATCM Console, the displays are arranged on the basis of the last used layout. If you have not yet created and saved a new layout, the standard layout of ATCM Console will be used for the arrangement of the displays.

Editing a Layout in the Layout Editor

To edit a layout, start the layout editor by clicking on the Edit button in the toolbar of ATCM Console or on the Edit Layout option in the Context menu of any display.

The arrangement of the displays which is shown in the layout editor always corresponds to the layout that is currently displayed in ATCM Console, even if the latter has not been saved yet. As you can see by comparing the two figures, one display corresponds to a row of squares with the same color and number in the layout editor. In the above figure, therefore, the first display from the left is represented in the layout editor by the gray squares with the number 1, the second display by the blue squares with the number 2 etc.

You can change the size and position of the displays in the layout editor. To do this, proceed as follows:

  1. On the very top of the layout editor, there is a row of numbered colored squares. Each square stands for a separate display. Click on one of these squares with the left mouse button. This square is now displayed similar to a pressed button; this is to signalize that it is enabled.
  2. The area underneath these squares represents the layout displayed in the Visualization view. The layout is divided into 4 x 4 squares. They form a grid indicating possible positions for the placement of a display. Click on one of the squares. This square is now displayed in the color and with the number that you have previously enabled by the mouse click on one of the squares in the upper row.
  3. To define the area which is to be covered by a display, click on additional squares so that the squares form a coherent area. The area covered later by a display in the Visualization view is determined by the squares with identical color and number. It is also possible to generate a display which consists of only one square.
  4. If your display has the desired size and position, click on another square in the upper row of squares to define the size and position of another display. All actions which you will from now on perform in the area underneath the row of squares, will be executed with the color and number of the enabled square.
  5. To close the layout editor and view the result of your arrangement, click OK. If, on the other hand, you do not want to keep the changes of your layout, click on Cancel.

Rules for Creating valid Layouts

Displays cannot be arranged at random in the layout editor. Therefore, you have to observe a few rules to create a layout which can be displayed as a valid layout in the Visualization view:

  • Squares with the same color and number have to be arranged in such a way as to form a coherent area since a display cannot be distributed over several places in the Visualization view which are not interconnected.
  • Displays may not be L-shaped, i.e. squares with the same color and number may not be arranged in L-shape.
  • Displays may not be arranged in circular shape (see Example of a Circular Arrangement).

Whether the current arrangement of displays results in a valid layout, is indicated in the layout editor by the text to the left of the two buttons OK and Cancel. If the current arrangement of displays results in a valid layout, the text reads Faceplate Layout: Valid. If, on the other hand, the current arrangement does not result in a valid layout, the text Faceplate Layout: Invalid is displayed. In this case, the OK button is disabled so that an invalid layout cannot be accepted in the Visualization view.

For further examples of valid and invalid layouts, see the section Examples of Display Layouts.

Editing Layouts via the Context Menu of the Displays

Each display has a Context menu by means of which size and position of the display can be influenced. With the Expand options, displays can be expanded to the left or right; with the Insert options, other displays can be inserted in the Visualization view. Changes which have been carried through via the Context menu are inserted in the currently enabled layout and displayed in the layout editor.
In addition, the Context menu contains two options for resetting the displayed object and the options Edit Layout and Copy Layout (on these options, see the sections Editing a Layout in the Layout Editor and Saving Altered Layouts).

The options in the Context menu are sensitive to context; i.e. only those options are displayed which are relevant for the current position of a display. If, for example, a display is already situated at the very left of the Visualization view, this display cannot be further expanded to the left. In this case, the option Expand Left does therefore not appear in the Context menu of the corresponding display. As a rule, a display can only be expanded in a direction where there are other displays as well. Similarly, the Insert options for displays are only shown if the maximum number of 4 displays in a vertical or horizontal row in the Visualization view has not yet been reached.

A display inserted via the Insert options will generally be empty. If, however, you have removed one or more displays from the Visualization view by the Expand option before inserting a display (see the section Expanding a Display to the left), the object which has been removed first and has no longer been displayed until the time of insertion is loaded in the inserted displays.

You can call the Context menu of a display by clicking with the right mouse button in the title bar of the desired display. Please note that the display-specific Context menu is only called if you click on the display title bar, whereas a click in the display area of the display opens a Context menu which contains options of the Internet Explorer.

With the Expand Left option, you can expand a display in this direction. The display which was located to the left of the selected display before this option was enabled, is no longer displayed. Instead, the selected display is expanded to cover the display area which was previously on its left.

Resetting the Display Content

With the option Clear in the Context menu, you can reset the display. The object previously represented in the selected display is no longer displayed and the display title bar reads Empty.

With the option Clear All, the content of all displays in ATCM Console is reset.

Saving Altered Layouts

Alterations carried through in a layout are not automatically saved and will therefore be lost as soon as you close ATCM Console or update the entire Visualization view. Therefore, you have to explicitly save a layout to maintain changes of this layout permanently.

The arrangement of displays currently represented in the Visualization view forms the basis for saving a layout. Both changes carried out via the layout editor and changes carried through by means of the Context menu are saved. If you have carried through changes in a layout and not yet saved them, the entry [Unsaved Layout] is displayed in the Layout list box of the ATCM Console toolbar.
During the saving of a layout, not only the current size and position of the displays are stored but also the linkage to the objects which are currently displayed. If you load a saved display, the objects assigned to the displays are therefore also visualized in the displays as long as there still is a valid linkage to the corresponding object. If the connection to the control on which the object was loaded no longer exists or if the object has been removed from the project, the message The object <object name> has been removed ! appears on the displays assigned to the corresponding objects.

To save an altered layout, proceed as follows:

  1. After editing a layout, click on the Disk button in the toolbar of ATCM Engineering

    - OR -

    click with the right mouse button on the colored title bar of any desired display to call the Context menu and in this menu, select the option Copy Layout.

  2. Enter the name of the new layout in the lower input field of the dialog. With the entry of the first character, the Save as button is enabled.



  3. Click on the Save as button to insert the layout in the Available Layouts list. From now on, the layout is available for selection with the new name in the list box of the ATCM Console toolbar.

  4. Click on Exit to close the Save Layout dialog. The saved layout automatically becomes the currently enabled layout.

You can save several layouts by calling the Save and Delete Layout dialog again after changing a layout and then entering a new name for the altered layout. As before, the layout is inserted in the Available Layouts list.
After saving, the last saved layout automatically becomes the currently enabled layout which is displayed in the Layout list box in the ATCM Console toolbar.

Saved layouts are managed inside ATCM Console and not stored as files. Therefore, it is not possible to copy layouts on floppy disk to use them on a different computer.

Loading a Stored Layout

If you have already saved one or more layouts, the currently loaded layout is displayed in the Layout list box in the ATCM Console toolbar.

To load a different layout,

  1. click in the Layout list box on the button with the downward arrow to scroll down the list of the list box. All stored layouts will be displayed in the list.
  2. Click on the desired layout with the left mouse button in order to load it. After a short time required by ATCMControl for restoring the Visualization view, the displays are shown in the selected layout.

Deleting a Layout

  1. Click on the Disk button to call the Save Layout dialog.

  2. All saved layouts are indicated in the Available Layouts list. Select the desired layout in the list. When you have selected a layout, the Delete button is enabled and the name of the layout is displayed in the lower box of the dialog. If you keep the CTRL or the Shift key pressed while selecting the layout, you can choose several layouts at one time.



  3. Click on Delete to delete the selected layout.

  4. Now click Exit to close the dialog.
