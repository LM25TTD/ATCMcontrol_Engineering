Variable quality

Variables of a project loaded on ATCM Control are read by ATCM Console within a defined cycle time and updated in the Visualization view. The connection for data exchange between ATCM Control and ATCM Console is established via the OPC interface. Each variable has its own OPC quality according to the OPC standard (good, bad or indeterminate). For instance, variables have the OPC quality good if there is a functioning connection to the data object loaded on a ATCM Control. If the connection is interrupted, the variable quality changes to bad.

The variable quality is visualized in the boxes for displaying the variable values. If the variable quality is good, the variable values are displayed in black color and normal font.
If the variable quality is bad or indeterminate, the variable values are displayed in red and crossed out.
