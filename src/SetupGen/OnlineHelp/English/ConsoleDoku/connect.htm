Connecting Possibilities / Starting ATCM Console

In ATCM Engineering, a configuration cannot only be assigned to the local computer but also to a remote computer. In ATCMControl, configuration and PC are practically identical after a configuration has been assigned to a PC. For execution, a configuration is loaded in the control of the computer to which the corresponding configuration was assigned. In this way, it is possible, for example, to compile a project in ATCM Engineering on the local computer and execute it on a remote computer by loading it in the control located in the remote computer. If you have a license for multiple configurations, you can also create several configurations in one project each of which is assigned to a different PC and therefore executed in the control of the corresponding PC.

In a started instance of ATCM Console, the process data of a specific configuration are visually displayed. For this purpose, ATCM Console establishes a connection, via the OPC interface, to the control of the computer assigned to the configuration. From the local computer, the configuration executed in the control of the local computer can be visually displayed and monitored in ATCM Console as well as a configuration which runs on a remote computer. It is also possible to monitor in ATCM Console, from several remote computers, the same configuration loaded on a ATCM Control.

The configuration to be displayed and the corresponding control are already selected when ATCM Console is started, i.e. the context from which ATCM Console was started determines which control the displayed data come from. There are three possibilities for starting ATCM Console:

  • Starting via the ATCM Tray in the Windows Program Menu
  • Starting from ATCM Engineering
  • Starting by Entering the Starting Address in the Address Field of the Internet Explorer

If there are several configurations in one single project, the configuration can still be changed after ATCM Console has been started.

If ATCM Console is started via the ATCM Tray or by entry of a starting address in the address field of the Internet Explorer, all objects contained in the project can be loaded in a Display via the Navigator of ATCM Console and be displayed there. The arrangement of the displays assigned to these objects can be changed via the Context menu or the Layout Editor (see section Working with Layouts). If, on the other hand, ATCM Console is started from ATCM Engineering, the selection of objects which can be loaded in a display depends on the calling position for ATCM Console (see Starting ATCM Console from ATCM Engineering).

Several instances of ATCM Console can be called at the same time, independent of the manner in which ATCM Console was started. To avoid problems when working with several instances of ATCM Console, you should check the check box Open browser window in a separate process in the Internet options of the Internet Explorer, in the Extended tab.

Starting ATCM Console via the ATCM Tray

If ATCM Console is started via the ATCM Tray, it is opened as a standalone application within the Internet Explorer. In a ATCM Console started via the ATCM Tray, the visualized process data always belong to the control on the local computer.

To start ATCM Console via the ATCM Tray,

double-click with the left mouse button on the ATCMControl icon on the right or bottom end of the Windows task menu

- OR -

click with the right mouse button on the ATCMControl icon to open its Context menu and select Open ATCM Console.

Starting ATCM Console from ATCM Engineering

If ATCM Console is started from ATCM Engineering, it is opened as a window within ATCM Engineering, i.e. the ATCM Console window can only be displaced within ATCM Engineering. From ATCM Engineering, several instances of ATCM Console can be called. Each instance is displayed in a separate window within ATCM Engineering.

Except for the Context menu, a Console window started from ATCM Engineering contains only the ATCM Control-specific operation and visualization elements. Part of the Internet Explorer functionality is replaced by a group of buttons with the most important navigation functions. For details on this group of buttons, see the section Internet Navigation Menu in ATCM Engineering.

From ATCM Engineering, ATCM Console is started via the Context menu of the ATCM Engineering navigator.
The Context menu of the Engineering Navigator with a menu item for opening ATCM Console can be called from the following positions:

in the Hardware view
from a PC icon
in the Instance view
from the icon of a configuration ,
a resource
or a task.

After selecting one of these icons, click on the right mouse button to call the Context menu and in this menu, select the option Open ATCM Console.

The control with which ATCM Console is connected when started from ATCM Engineering, and the visualized object of the configuration depend on the calling position which was selected for opening the Context menu.

If only one configuration has been generated in the project currently opened in ATCM Engineering, the visualized process data are always the ones loaded on the control which is located on the PC assigned to the configuration. This can be either the local PC or a remote computer. All objects in the configuration are available for selection in the Navigator of ATCM Console if the calling position for the Context menu is either the PC icon or the configuration icon. If the Context menu is called from the position of a resource icon, the task list of the configuration is displayed; if it is called from the position of a task icon, the faceplate of the previously selected task is displayed (for more information on the Navigator and the faceplates in ATCM Console, see the sections User Interface and Displays). If called from the last two positions, the Navigator is not shown; as a result of this the selection of objects is restricted.

If you have a license for multiple configurations, you can also create several configurations in one project. In this case, ATCM Console is connected to the control on the PC whose assigned configuration (or an icon under this configuration) has been selected as calling position for the Context menu. Starting ATCM Console by Entering the Starting Address in the Internet Explorer

If ATCM Console is started from the Internet Explorer, the PC to whose control ATCM Console is to be connected can be indicated together with the starting address:

  1. Start the Internet Explorer.
  2. Enter the desired starting page in the address box. The address must have the following syntax:
    'http//<PC name>/4C'. If your PC is called, for instance, Automation1, the address is 'http//Automation1/4C'.
  3. Click on Change to or press the Enter key to call the entered address.

Password Protection

In ATCM Engineering, a password for the currently loaded project can be entered (see also the section "Project-specific password"). This password becomes effective after compilation of the project. If a configuration contained in a password-protected project is loaded on a ATCM Control, a password inquiry is carried through when ATCM Console is started. For password-protected projects, a password must be entered independently of whether the control to be connected is on a local or a remote computer. However, the inquiry is only carried through if ATCM Console is started outside of ATCM Engineering, i.e. via the ATCM Tray or the address field of the Internet Explorer.

The security check differentiates between two access levels:

  • Read only access: The console can be started and the process variables monitored. The restriction of the read only access is that values of process variables cannot be changed. Tasks and resources cannot be started or stopped, either.
  • Unlimited access: Values of process variables can be read and changed if there is write access to the respective variable from the assigned display. Tasks and resources can be started and stopped.

Which access level is valid for a configuration is determined when the password is assigned in ATCM Engineering (in the Tools - Project Settings... menu, Console tab).

Please note that a password inquiry also takes place if, for example, you update the entire Visualization view of ATCM Console after a download via the Update button in the toolbar of the Internet Explorer or if you update a specific display via the Explorer Context menu of the display. If the latter is the case, the password dialog is only displayed in the updated display.

Logging in with ATCM Console

To log in with ATCM Console in case of a password protection, write the password in the input field and confirm with Enter.
If you have misspelled the password, you can clear the input field by clicking on Reset and repeat the entry.

If you enter an invalid password, access to the console is denied and the Login dialog box is restarted.
If, on the other hand, you enter a valid password, a console session is started with all access rights which are made available by the entered password.

Interrupting a Connection to ATCM Control

An existing connection is interrupted if one of the following events occurs:

  • The Logout button of the console is pressed.
  • A new project is loaded on the control connected to ATCM Console.
  • The connection is interrupted, for instance, by closing of the Internet Explorer, and not restarted within approx. 30 seconds. In this case, the data of the current connection, which have been stored on the server, are lost.

The Logout button is located in the toolbar of ATCM Console. It is only displayed if the loaded configuration is protected by a password. If the currently loaded configuration has no password, this button is not visible.
With a click on the Logout button, the existing connection is interrupted and the Login dialog redisplayed.
If, after logging out, you log back in with the same password, you will reestablish a connection to the control to which you were connected before.
