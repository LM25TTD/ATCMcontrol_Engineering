This chapter will show you how to manage projects. You will learn how to create, open, close, save, print and delete a project and how to generate, download and debug it.

Create a new Project

You can create a new project by going to the File menu and clicking on New Project

- OR -

by clicking on the New Project button in the tool bar.

Follow the instructions of the Wizard to create a new project.

Open a Project

You can open a project by going to the File menu and clicking on Open Project

- OR -

by clicking on the Open Project button in the tool bar.

Close a Project

If you want to close the current project, go to the File menu and click on Close Project.
The project is closed.

Save the Current Project

Save

You can save the current project by going to the File menu and clicking on Save Project

- OR -

by clicking on the Save Project button in the tool bar.
The project is saved.

Delete the Current Project

You have to open a project before you can delete it.
Then click in the File menu on Delete Project. Acknowledge the safety inquiry with Yes to delete the project.

All files of this project are deleted.

Convert Projects Created with an Earlier ATCMControl Version

Due to the numerous changes and new features in the current ATCMControl version, it has been necessary to substantially rearrange the design of the ATCMControl project file '*.4cp'. Projects which were created with one of the ATCMControl predecessor versions can therefore no longer be loaded with the current version. Naturally, though, it has been ensured that you can continue to work with your old projects without having to carry through any changes in the project yourself. Together with the new ATCMControl version, the conversion file 4Convert.exe responsible for the convenient conversion of older projects has been installed in the 'Eng' directory directly underneath your ATCMControl directory. This file is automatically called if you try to load a project created with a predecessor version of V1.2.

Therefore, you can directly convert a project in ATCM Engineering that was created with an earlier version than V1.2:

  1. Start ATCM Engineering and call the dialog box for opening a project with the Open Project option.

  2. In this dialog box, select the desired project file. You can recognize a project file by the file ending '.4cp'.



    Click on the Open button.

  3. A message box is displayed which informs you that you are trying to load a project created with an earlier ATCMControl version and which asks you whether you want to convert the project to the new format.



    Confirm this message with Yes. Your project is now converted to the new format.

  4. Messages about the conversion progress are displayed in a DOS box.



    During the conversion, a backup directory is created which contains your old project in the unchanged format. The backup directory is part of the same higher-level directory as your original project directory. The name of the backup directory is made up of the name of your old project directory with the ending BACKUP, separated from the name by an underline character, and the date and time when the directory was created. If the directory with your old project file was called Pump, for instance, the backup directory might be called Pump_BACKUP_Wed_Jul_21_16-38-39_1999. The new project, on the other hand, is in the directory where your old project was before the conversion, i.e. in our example in the Pump directory. You can also see the name of the backup directory in the first line of the DOS box.
    After the message conversion completed successfully has been displayed in the DOS box, press the Enter key to close the box and finish the conversion process.

  5. Open the converted project which is now located in the directory of the original project in order to load it in ATCM Engineering and make sure that it has been converted correctly.

Instead of converting an old project in ATCM Engineering, you can also call 4Convert.exe in the DOS box or in the Windows instruction menu.

This is how you can call 4Convert.exe in the DOS box:

  1. Click on Input Request in the Windows Start menu .
  2. Go to the Eng directory in your ATCMControl directory.
  3. For parameters, 4Convert.exe needs the path and the file name of the project to be converted. Therefore, enter 4Convert and then the path and 4cp file of your project, e.g. 4Convert C:\Programs\4Control\Projects\pump.4cp.
  4. Confirm your input by pressing the Enter key. As has been described in point 4, your project is now converted to the new format and a backup directory with your original project is created.
  5. After the message conversion completed successfully has been displayed in the DOS box, press the Enter key to close the box and finish the conversion process.

If you want to call 4Convert.exe via the instruction menu, proceed as follows:

  1. Click on Execute... in the Windows Start menu . .
  2. Enter the complete path of 4Convert.exe and, separated by an empty space, the complete path of the project file to be converted.
  3. Confirm your input with OK. The DOS box is opened and messages on the conversion progress are issued. During conversion, a backup directory with your old project is created.
  4. After the message conversion completed successfully has been displayed in the DOS box, press the Enter key to close the box and finish the conversion process.

Generate the Project

You must assign every configuration (default: Configuration.CON) to a PC before you can generate the project.
  1. Click on the Hardware tree in the Navigator view.
  2. Click on the 4CProject icon with the right mouse button to open the Context menu.
  3. Select Insert PC....
    The Add PC dialog is displayed.
  4. Enter the name of the PC (if necessary, look in the network area of the NT system control) and confirm it with OK.
  5. Go to the icon of the desired PC.
  6. Click on it with the right mouse button to open the Context menu.
  7. Select Assignment.... .
    The Assignment dialog appears.
  8. Select the configuration which you want to assign to the PC and click OK.
    This configuration has now been prepared for continuation of the procedure.

The efforts in generating the code with ATCM Engineering have been reduced to a minimum. If you want to display all error messages,

go to the Build menu and click on Build All

- OR -

click on the Build All button in the tool bar.

The Message view is displayed in the lower part of the ATCM Engineering window; all error messages and warning signals are displayed here.
For a download, the project must have been compiled without any errors.

Download the Configuration of the Current Project

As soon as you have generated the code, you can download it very easily on ATCM Control. To do this, proceed as described in the following.
Depending on how many configurations belong to the current project and how many are to be downloaded simultaneously on the PCs, you can download or update different numbers of configurations on the respective ATCM Controls.

During project compilation, the compiler generates domains of each object. These domains are data packages containing information on the source code and the visual display of the object. The domains are displayed in the Selective Update dialog.

In the standard setting, the configuration is started directly after the download.

Download 

All configurations

If you want to download all configurations on all ATCM Controls in one single step,

go to the Build menu and click on Download/All Configurations

- OR -

click on the All Configurations button in the tool bar.

If there is no debug connection to ATCM Control yet, a dialog box appears on your screen to ask you whether such a connection is to be established.
During a debug connection, only one ATCM Engineering has access to ATCM Control.

Click Yes if you want to download a new project.

ATCM Engineering now begins to establish a debug connection.

You can abort this process by clicking on the Cancel button.

If the debug connection cannot be established, e.g. because another user has already established one, the following message appears after some time:

In this case, click on Yes to interrupt the existing connection.

You can change the time-out, i.e. the period of time during which a connection is attempted before an error message appears, in the tab of the Options dialog box. You can call the Options dialog box by clicking on Project Settings in the Tools menu.

When a debug connection has been successfully established, the control code of all ATCM Controls is deleted and all configurations of the current project are downloaded simultaneously on all ATCM Controls.

Selected configurations

To download an individual configuration on the assigned ATCM Control, go to the Build menu, click on Download and select the respective ATCM Control in the menu. The Message view at the bottom of the screen changes to the Download tab and displays the control messages.

The control code of the ATCM Control is deleted and the assigned configuration is downloaded on the selected ATCM Control.

Download Changes

All configurations

If you want to download the differences between the code of the running ATCM Control and the current status of the source code on your ATCM Engineering, you can update the running ATCM Controls.
Other than Download All, this function takes into account only the differences between the data on ATCM Control and the current project on your ATCM Engineering. After the generation, ATCM Control is only updated in terms of the differences. This helps you save time during the download. All altered or new data are transferred to all ATCM Controls.
Go to the Build menu and click on Download Changes / All Configurations.
If there is no debug connection to ATCM Control yet, a dialog box appears on your screen to ask you whether such a connection is to be established.
During a debug connection, only one ATCM Engineering has access to ATCM Control.

Click Yes if you want to download a new project.

ATCM Engineering now begins to establish a debug connection.

You can abort this process by clicking on the Cancel button.

As soon as the connection has been established successfully, updating starts.

Selected configurations

If you want to download the differences between the runtime code of an individual ATCM Control and the current status of the project on your ATCM Engineering, you can update each running ATCM Control separately.
Other than Download/All Configurations, this function takes into account only the differences between the data on the selected ATCM Control and the current project on your ATCM Engineering. After the generation, ATCM Control is only updated in terms of the differences. This helps you save time during the download. All altered or new data are transferred to the selected ATCM Control.
Go to the Build menu and click on the desired ATCM Control under Download Changes / Selected Configurations.

The updating process begins directly after.

Download Selection

Configuration survey

In the Download Overview dialog box, you can monitor the download status of all ATCM Controls of the project. You can call the dialog box by going to the Build menu and clicking on Configuration Overview. All ATCM Controls are listed with their names. You can obtain information on the name of the downloaded project, the last download, the user who last downloaded the project and the current status.

It is possible to view the differences between the data of the project on your ATCM Engineering and the content of ATCM Control by selecting the desired ATCM Control and clicking on Details.
The Selective Update dialog is displayed.

Selective update

All domains of the project are listed with their names and types.
ATCMControl compares all domains on ATCM Control with all domains of the current project on your ATCM Engineering. The differences are displayed in this dialog. In the Original Action to do column, you can obtain information on the currently proposed status of each domain on ATCM Control. This status is meant only as a proposal for each domain. It is, however, the action which ATCMControl would carry through next for the respective domain. If you click on Start Download, you will download all domains of the project on ATCM Control as they are proposed.
You can also ignore these proposals, though,and determine precisely what is supposed to happen with the selected domain.
To do this, click on the domain with the right mouse button.

Multiple selections are possible! Hold the Shift key down and select the desired domains.
If several domains are selected, the Context menu which is displayed after a click with the right mouse button refers to all selected domains.

Debug the Project

Debugging a project helps you to find errors in your code. For debugging, you can set breakpoints and execute the code line per line in the Single Step Run.

The use of breakpoints and Single Step Run is possible in the text editor as well as in the graphical editor.

You can only debug your code if a debug connection has been established. During a debug connection, only one ATCM Engineering may be connected to ATCM Control. To establish a debug connection, click on Enable Debug in the Control menu and select the desired configuration (the PC) .

Simple / advanced debugging

Please note that intensive debugging affects the execution time of your code. For this reason, two debug modes are available in ATCMControl. You can determine the mode in the Debug tab in the Options dialog box. You can call this dialog box via the Project Settings option in the Tools menu.

After you have changed the debug mode, you should recompile and download the project.

Whereas the Simple Debugging mode does not sensibly impair the runtime behavior of the control, the Advanced Debugging mode reduces system performance by approx. 30 %. The following differences exist between the two debug modes:

  • Simple debugging: Breakpoints are permitted at the beginning or end of a program or function block. In the ST code, these places are defined by the first statement (i.e. the first line in the code which is concluded by a semicolon after the variable declaration) and the END_PROGRAM or END_FUNCTION_BLOCK statement. In the FBD editor, specific places for the beginning and the end of a program or function block are defined in the upper left and lower right corners.



  • Advanced debugging: In this mode, breakpoints are permitted in all statements . Since this debug mode uses a large proportion of the system resources, system performance is reduced by approx. 30%. During compilation of a project whose debug mode has been set at Advanced Debugging, a message is therefore issued in the Message view that the project is compiled with advanced debug settings.

Set/remove breakpoint

Breakpoints can be set for program instances and function blocks.
To set a breakpoint,

position the cursor in the source code editor in the place where you want to set the breakpoint and click on Set/Remove Breakpoint in the Context menu

- OR -

position the cursor in the source code editor in the place where you want to set the breakpoint and press the F9 key.

The breakpoint is set in the editor in the nearest possible place to the current cursor position. You can recognize a set breakpoint by the red circle in the editor frame.

To remove a breakpoint,

position the cursor in the editor near the breakpoint that you want to remove and click on Set/Remove Breakpoint in the Context menu

- OR -

place the cursor in the editor near the breakpoint that you want to remove and press the F9 key.

 

To remove all breakpoints,

click on Remove All Breakpoints in the Context menu

- OR -

press the key combination Ctrl-Shift-F9.

Working with breakpoints

Breakpoints are represented in the editors by the following icons:

enabled breakpoint before the program execution is stopped at this position
enabled breakpoint at which the program execution was stopped
disabled breakpoint ignored during debugging

To enable or disable a breakpoint, click on Enable/Disable Breakpoint in the Context menu

- OR -

press the key combination Ctrl-F9.

After the breakpoint is reached, the running task changes to the Break status. There are two possibilities to continue program execution:

Continue program execution to the next breakpoint. For this purpose, click in the Context menu on Continue or press the F5 key.

Execute the program up to the next place where a breakpoint is permitted (this place depends on whether advanced or simple debugging has been set). For this purpose, click in the Context menu on Single Step or press the F10 key.

Exit ATCM Engineering

To exit ATCM Engineering, go to the File menu and click on Exit.

All changes which have not been saved are saved now, ATCM Engineering is closed and all connections to ATCM Control are interrupted.
