Aufbau

ATCM Console ist eine in den Internet Explorer eingebettete Anwendung. Die grafische Oberfläche von ATCM Console wird mit Hilfe von HTML, ASP, Java Script und VB Script realisiert und als HTML-Seite im Fenster des Internet Explorers dargestellt. Prinzipiell stehen Ihnen daher bei der Arbeit mit ATCM Console alle Bedienelemente und die gesamte Funktionalität des Internet Explorers zur Verfügung. Beim Start von ATCM Console aus ATCM Engineering enthält das Konsolenfenster mit Ausnahme des Kontext-Menüs allerdings nur die ATCM Control-spezifischen Bedien- und Visualisierungselemente. Ein Teil der Internet Explorer-Funktionalität wird durch eine Schaltflächengruppe ersetzt, die die wichtigsten Navigationsfunktionen zur Verfügung stellt. Erläuterungen zu dieser Schaltflächengruppe finden Sie im Abschnitt Internet-Navigationsleiste in ATCM Engineering. Zur Navigation in ATCM Console über die Internet-Funktionen siehe In der Visualisierungs-Ansicht navigieren.

Die eigentliche Bedienoberfläche von ATCM Console beginnt unterhalb der Menü- und Symbolleiste des Internet Explorers und gliedert sich in 4 Bereiche:

  • Navigator-Ansicht (linke Seite der Bedienoberfläche)
  • Visualisierungsbereich (rechte Seite der Bedienoberfläche, ist meist in mehrere Anzeigemodule unterteilt)
  • Werkzeugleiste
  • Statuszeile

Die Visualisierungs-Ansicht

Zusammenarbeit von Visualisierungs- und Navigator-Ansicht

Visualisierungs- und Navigator-Ansicht hängen aufs Engste miteinander zusammen. Während in der Visualisierungs-Ansicht auf ATCM Control geladene Prozeßdaten angezeigt und gegebenenfalls grafisch dargestellt werden, wird über die Navigator-Ansicht geregelt, welche Daten in der Visualisierungsansicht zu sehen sind. Entsprechend der Objekthierarchie des ATCMControl-Programmiermodells können Daten Objekten (z.B. Funktionsbausteinen, Programmen, Tasks, Ressourcen) zugeordnet und in Einheiten zusammengefaßt werden. Dementsprechend werden auch von ATCM Control gelieferte Prozeßdaten zur Darstellung in der Visualisierungs-Ansicht in optische Einheiten zusammengefaßt, die diesem Programmiermodell entsprechen. Die Auswahl des in einer Visualisierungseinheit dargestellten Objekts erfolgt über die Navigator-Ansicht, die die im Projekt enthaltenen Objekte gemäß der Hierarchie des IEC-Programmiermodells in Baumstruktur darstellt. Jedes Objekt in der Baumstruktur hat Hyperlinkcharakter, sodaß ein Klick auf ein Objekt in der Navigator-Ansicht zur Darstellung in einer Visualisierungseinheit der Visualisierungs-Ansicht führt.

Eine einzelne Visualisierungseinheit, die wir im folgenden als Display bezeichnen werden, erkennen Sie an dem grauen Rahmen, der jedes Display umgibt. Größe und Position der Displays können verändert werden. Wenn Sie ATCM Console zum ersten Mal starten, sind die Displays zunächst noch leer, und die Kopfleisten der Displays zeigen dementsprechend die Beschriftung Empty.

Objekte in einem Display laden und visualisieren

Um in einem der noch leeren Displays die Prozeßdaten eines Objekts zu laden, klicken Sie in der Navigator-Ansicht auf ein Symbol, hinter dem sich eine blaue und unterstrichene Bezeichnung befindet. Statt auf das Symbol können Sie auch auf die Bezeichnung klicken. Die blaue Farbe und die Unterstreichung signalisieren, daß es sich bei dem entsprechenden Objekt um einen Link handelt, bei dessen Aktivierung die zu dem angeklickten Objekt gehörenden Prozeßdaten in dem aktuell aktiven Display angezeigt werden.
Welche Objekte in der Navigator-Ansicht für die Visualisierung zur Verfügung stehen und in welcher Hierarchie diese angeordnet sind, erfahren Sie im Abschnitt Navigator-Ansicht.

Die Prozeßdaten eines im Navigator angeklickten Objekts werden immer in das aktuell aktive Display geladen. Das aktuell aktive Display erkennen Sie an der gelben Farbe der Kopfleiste, während die Kopfleisten von inaktiven Displays die Farbe blau besitzen. Noch während des Ladens der Daten wird das nächste Display aktiv, wogegen das zuvor aktive Display wieder in den Status inaktiv wechselt. Das bedeutet, daß beim nächsten Klicken auf ein Objekt im Navigator dessen Prozeßdaten jetzt in das nachfolgende Display geladen werden. Objekte werden grundsätzlich in der Richtung von links nach rechts und bei mehreren Reihen von oben nach unten, wenn das letzte Display einer Reihe erreicht ist, eingefügt.

Sie müssen sich allerdings nicht an diese Standardreihenfolge halten. Sie können ein Display auch gezielt aktivieren und damit festlegen, in welches Display ein Objekt eingefügt werden soll.

So fügen Sie ein Objekt in ein von Ihnen frei gewähltes Display ein:

  1. Klicken Sie mit der linken Maustaste auf die Kopfleiste des gewünschten Displays. Die Kopfleiste signalisiert durch Farbänderung, daß das Display jetzt aktiv ist.
  2. Klicken Sie auf das gewünschte Objekt in der Navigator-Ansicht von ATCM Console. Das selektierte Objekt wird in das aktuell aktive Display eingefügt. Die Kopfleiste des Displays zeigt den Namen des Objekts an.

Auf die oben beschriebene Weise können Sie auch dem Display ein anderes Objekt zuweisen, wenn in dem entsprechenden Display bereits Objektdaten visualisiert werden. Aktivieren Sie dazu das einfach das gewünschte Display per Mausklick auf dessen Kopfleiste, und wählen Sie ein anderes Objekt in der Navigator-Ansicht aus.

Displays können nicht nur innerhalb von ATCM Console, sondern auch in einem eigenen Fenster außerhalb von ATCM Console dargestellt werden.
Um ein Display in einem eigenen Fenster darzustellen, klicken Sie in der Navigator-Ansicht auf das gewünschte Objekt, während Sie die STRG-Taste gedrückt halten. Das Display wird jetzt als eigenes Fenster dargestellt.

Die Anordnung und Größe der Displays kann mit Hilfe des Layout Editors und des Kontext-Menüs verändert werden. Sie dazu das Kapitel Layouts.

Die Navigator-Ansicht

Über die Navigator-Ansicht wird die Anzeige der Displays in der Visualisierungs-Ansicht gesteuert. Sie stellt die in ihrem Projekt enthaltenen Objekte (Funktionsbaustein-Instanzen, Programme, Tasks usf.) in Baumstruktur dar. Alle Objekte werden mit einem Symbol und dem Namen des Objekts angezeigt. Die meisten Objektnamen sind unterstrichen und werden in blauer Farbe dargestellt. Die Objektnamen signalisieren damit, daß es sich bei diesen Repräsentationen der Objekte um Hyperlinks handelt, über die die Daten des ausgewählten Objekts in einem Display geladen und visualisiert werden können.

Um die Daten eines Objekts in einem Display zu laden gehen Sie folgendermaßen vor:

  1. Aktivieren Sie in der Visualisierungs-Ansicht das Display, in dem das Objekt dargestellt werden soll, indem Sie in dessen Kopfleiste klicken. Wenn Sie ein Display nicht explizit aktivieren, beziehen sich alle nachfolgenden Aktionen auf das Display, das nach der von der Visualisiungsansicht vorgegebenen Standardreihenfolge aktiv ist (siehe Objekte in einem Display laden und visualisieren).
  2. Klicken Sie in der Navigator-Ansicht auf das gewünschte Objekt. Das Objekt wird in dem zuvor aktivierten Display angezeigt.

Die Navigator-Ansicht gliedert sich in die Registerkarten Project und Function Blocks, in denen die Objekte des auf ATCM Control geladenen Projekts jeweils in einer Baumstruktur dargestellt sind. Die Organisation der Baumstruktur unterscheidet sich in beiden Registerkarten.

Die Registerkarte Project

Die Projekt-Ansicht entspricht im Wesentlichen der Instance-Ansicht des Navigators von ATCM Engineering. In beiden Ansichten orientiert sich Darstellung der in Baumstruktur angeordneten Objekte an der Struktur des Programmiermodells von ST bzw. FBD. Ebenen der Baumstruktur leiten sich aus der Objekt- bzw. Aufrufhierarchie eines Projekts ab. Bspw. können Tasks Programm-Instanzen aufrufen, die ihrerseits Funtionsbaustein-Instanzen aufrufen usf. Dementsprechend wird jedes aufgerufene Objekt eine Ebene unterhalb des aufrufenden Objekts dargestellt. Im folgenden wird eine Übersicht der Baumstruktur organisierten Objekte gegeben. Einzelheiten zur Visualisierung dieser Objekt in den Displays finden Sie im Kapitel Displays.

Zuoberst der Baumstruktur befindet sich das Projektsymbol mit dem Projektnamen. Da nur die im Projekt enthaltenen Objekte, nicht jedoch das Gesamtprojekt in der Visualisierungsansicht dargestellt werden kann, hat das Anklicken des Projektsymbols oder -namens keine Auswirkung auf die Darstellung in der Visualisierungsansicht.

Direkt unterhalb des Projektsymbols befindet sich immer die Configuration List. Diese hat keine Entsprechung in der Instanz-Ansicht von ATCM Engineering. Falls Sie über eine Multikonfigurations-Lizenz verfügen, kann ein Projekt mehrere Konfigurationen enthalten. Die Configuration List gibt einen Überblick über alle in einem PC enthaltenen Konfigurationen, sofern diese auf ein Control geladen sind. Ist zwar eine gültige Konfiguration im Projekt angelegt und auch einem PC zugeordnet, die Konfiguration jedoch nicht auf das zugeordnete Control geladen worden, wird diese Konfiguration nicht in der Liste aufgeführt.

Die in einem Projekt enthaltenen Konfigurationen befinden sich in der Baumstruktur auf der gleichen Hierarchieebene wie die Configuration List, jedoch immer unterhalb der Configuration List. In der Baumstruktur kann immer nur eine Konfiguration mittels Mausklick auf das Plussymbol expandiert werden, während alle anderen Konfigurationen nicht über ein Plussymbol verfügen. Stattdessen sind die anderen Konfigurationen durch blaue Farbe und Unterstreichung als Hyperlink gekennzeichnet. Ein Klick auf den Link führt in diesem Fall allerdings nicht wie gewohnt zu einer Anzeige der Konfiguration im aktuell aktiven Display, sondern das Plussymbol zur Expandierung einer Konfiguration wechselt zu der angeklickten Konfiguration. Damit können jetzt die unter dieser Konfiguration enthaltenen Objekte in der Baumstruktur expandiert und in ein Display geladen werden, dafür sind allerdings die zu der vorhergehenden Konfiguration gehörenden Objekte jetzt unsichtbar.

Unterhalb der Konfiguration wird die der Konfiguration zugeordnete Ressource angezeigt. Ein Klick auf diesen Hyperlink zeigt im aktiven Display Informationen zum Laufzeitstatus der Ressource an.

Einer Ressource sind ein oder mehrere Tasks zugeordnet. Auf der nächsten Ebene unterhalb der Ressource befindet sich daher die Task List, die eine Liste aller der Ressource zugeordneten Tasks enthält.

Die Message History unterhalb der Task List enthält eine Liste der ATCM Control- und benutzerdefinierten Meldungen, die sich auf die Ressource, unterhalb der sich die Message History befindet, beziehen.

Die nächste Ebenen bilden die in der Task List aufgeführten Tasks. Da eine Ressource mehrere Tasks enthalten kann, können auf dieser Ebenen mehrere Tasks in der Baumstruktur untereinander angezeigt werden.

Tasks können wiederum mehrere Programme zugeordnet werden. Jede Task kann deshalb expandiert werden. Nach der Expansion sind in der Baumstruktur eine Ebene unterhalb der Task die von der Task aufgerufenen Programminstanzen zu sehen.

Programme können ihrerseits wiederum Funktionsbaustein-Instanzen aufrufen. Die von einem Programm aufgerufenen Funktionsbaustein-Instanzen werden daher bei Expansion des Programm-Zweigs auf der nächsten Ebenen angezeigt.
Funktionsbaustein-Instanzen bilden die letzte Ebene innerhalb der Baumstruktur. Da Fuktionsbaustein-Instanzen allerdings wieder andere Instanzen von Funktionsbausteinen oder Strukturen aufrufen können, kann ab hier die Baumstruktur gemäß der Aufrufhierarchie beliebig tief verzweigen, je nach dem, wie stark die Funktionsbaustein-Aufrufe geschachtelt sind.

Instanzen von Strukturen werden eine Ebene unterhalb der Funktionsbaustein-Instanz angezeigt, in der die jeweilige Struktur-Instanz deklariert wurde.

Die Registerkarte Function Blocks

Die Funktionsbaustein-Ansicht ist eine reduzierte Version der in der Projekt-Ansicht dargestellten Projektstruktur. Hier werden auschließlich die in einem Projekt enthaltenen Funktionsbaustein-Instanzen in Baumstruktur angezeigt. Die Gliederung der Ebenen in der Baumstruktur folgt wie in der Projektansicht der Aufrufhierarchie eine Projekts. Aufrufende Funktionsbaustein-Instanzen befinden sich in der Baumstruktur immer eine Ebene höher als aufgerufene Funktionsbaustein-Instanzen. Aufrufende Funktionsbaustein-Instanzen sind daher über das Plus-Symbol expandierbar. Unterhalb der expandierten Funktionsbaustein-Instanz werden dann die aufgerufenen Funktionsbaustein-Instanzen angezeigt.
Angezeigt werden sowohl Instanzen von benutzerdefinierten Funktionsbausteinen als auch von Funktionsbausteinen aus Standardbibliotheken.
Instanzen von Strukturen werden eine Ebene unterhalb der Funktionsbaustein-Instanz angezeigt, in der die jeweilige Struktur-Instanz deklariert wurde.

Die Statuszeile

Die Statuszeile gliedert sich in drei Abschnitte:

Linker Abschnitt:
Hier werden alle während der Arbeit mit ATCM Console auftretenden Fehlermeldungen angezeigt.
Mittlerer Abschnitt:
Hier wird nach einem Download eines in ATCM Console visualisierten Objekts die blinkende Meldung DOWNLOAD in roter Farbe angezeigt. Die Meldung signalisiert, daß nach einem Download die angezeigten Daten evtl. nicht mehr dem auf ATCM Control geladenen aktuellen Stand entsprechen. Nach der Aktualisierung des Visualisierungsbereichs wird die Meldung wieder ausgeblendet.
Siehe auch Navigator- und Visualisierungs-Ansicht aktualisieren.
Rechter Abschnitt:
Hier wird der Gültigkeitszeitraum der ATCM Control-Lizenz angezeigt. Beträgt der verbleibende Gültigkeitszeitraum weniger als 3 Tage, wird die Anzeige in roter Farbe dargestellt. Beträgt der Gültigkeitszeitraum nur noch einen Tag, beginnt die Anzeige zu blinken.

Die Internet-Navigationsleiste in ATCM Engineering

Beim Start von ATCM Console aus ATCM Engineering wird das Konsolenfenster ohne die Menü- und Werkzeugleiste des Internet Explorers geöffnet. Ein Teil der Funktionalität des Internet Explorers wird deshalb über die Schaltflächen auf der Internet-Navigationsleiste ersetzt, die sich auf der Symbolleiste von ATCM Engineering befindet (siehe auch Abschnitt Die Internet-Funktionen). Wenn kein Fenster von ATCM Console in ATCM Engineering geöffnet ist, sind diese Schaltflächen deaktiviert. Sind dagegen mehrere Fenster von ATCM Console geöffnet, beziehen sich die Funktionen der Schaltflächen immer auf das sich im Vordergrund befindende Konsolenfenster.

Die Funktionen der Schaltflächen sind in der folgende Tabelle aufgelistet:

Zurück zur vorhergehenden Seite
Vorwärts zur nächsten Seite
Abbrechen des Ladevorgangs
Seite aktualisieren
Zurück zur Home-Site
Netz durchsuchen

Beachten Sie beim Einsatz der beiden Schaltflächen zur Rückwärts- und Vorwärts-Navigation bitte folgendes:
Beide Schaltflächen beziehen sich auf das aktuell aktive Display von ATCM Console. Wenn Sie im Verlauf Ihrer Arbeit mit ATCM Console mehrere Objekte in dasselbe Display geladen haben, können Sie mit diesen Schaltflächen durch die zuvor geladenen Objekte in der Reihenfolge des Ladevorgangs vor- und zurücknavigieren.

Die Internet-Funktionen

ATCM Console ist eine in den Internet-Explorer eingebettete Anwendung. Daher stehen Ihnen bei der Arbeit mit ATCM Console prinzipiell auch alle Funktionen zur Verfügung, die Ihnen der Internet Explorer ohne ATCM Console bietet. Einige dieser Funktionen können Sie zur Steuerung der Navigator- und Visualisierungs-Ansicht nutzen. In diesem Abschnitt nicht dokumentierte Funktionen des Internet Explorers schlagen Sie bitte in der Hilfe zum Internet Explorer nach.

Navigator- und Visualisierungs-Ansicht aktualisieren

Die in der Visualisierungs-Ansicht dargestellten Objekte werden innerhalb einer definierten Zykluszeit automatisch aktualisiert. Sind nach einem vollständigen oder inkrementellen Download allerdings neue Objekte oder Variablen im Projekt angelegt worden, werden diese neu hinzugekommenen Elemente nicht automatisch in ATCM Console angezeigt. Bspw. werden Variablen, die vor dem Download innerhalb eines Funktionsbausteins neu angelegt wurden, nach dem Download nicht automatisch in dem Display angezeigt, das den entsprechenden Funktionsbaustein darstellt. Ebensowenig werden neu hinzugekommene Objekte in der Navigator-Ansicht automatisch in die Baumstruktur eingefügt. Um Sie auf diese Tatsache aufmerksam zu machen, wird nach einem neuen Download in der Statuszeile die rot blinkende Meldung DOWNLOAD eingeblendet. Damit Sie auch die nach dem Download neu geladenen Elemente in ATCM Console beobachten zu können, müssen Sie die Ansichten von ATCM Console aktualisieren. Nach der Aktualisierung verschwindet auch die DOWNLOAD-Meldung wieder.
Gelöschte Variablen werden nach einem Download nicht aus dem Display entfernt, sondern signalisieren die OPC-Qualität schlecht (siehe auch Abschnitt Variablenqualität ) . Um die nach dem Download gültige Datenorganisation des in ein Display geladenen Objekts anzuzeigen, müssen Sie das entsprechende Display oder die gesamte Visualisierungs-Ansicht manuell aktualisieren.
Wenn Sie ein ganzes Objekt (z.B. Funktionsbaustein, Task) vor einem Download aus dem Projekt entfernen, wird nach dem Download in dem diesem Objekt zugeordneten Display die Meldung The object <object name> has been removed ausgegeben. Ordnen Sie in diesem Fall dem entsprechenden Display einen neuen gültigen Baustein zu.

Um sowohl die Navigator- als auch die gesamte Visualisierungs-Ansicht zu aktualisieren, klicken Sie auf die Schaltfläche Aktualisieren auf der Internet Explorer-Werkzeugleiste, falls Sie ATCM Console außerhalb von ATCM Engineering gestartet haben,

- ODER -

klicken Sie auf die Schaltfläche zur Seitenaktualisierung auf der Internet-Navigationsleiste von ATCM Engineering, falls Sie ATCM Console aus ATCM Engineering heraus gestartet haben.

Neben der Aktualisierung der gesamten Navigator- und Visualisierungs-Ansicht haben Sie auch die Möglichkeit, einzelne Displays zu aktualisieren. Die Download-Meldung wird nach der Aktualisierung eines einzelnen Displays allerdings nicht entfernt.

So gehen Sie vor, um ein einzelnes Display zu aktualisieren:

  1. Klicken Sie mit der rechten Maustaste in den Anzeigebereich des gewünschten Displays, um das Kontext-Menü zu öffnen. Das aufgerufene Kontext-Menü ist ein Menü des Internet Explorers. Die im Menü aufgeführten Optionen beziehen sich auf das selektierte Display.
  2. Klicken Sie im Kontext-Menü auf die Option Aktualisieren. Da sich das Kontext-Menü auschließlich auf das selektierte Display bezieht, wird nur dieses Display aktualisiert.

In der Visualisierungs-Ansicht navigieren

Wenn Sie die in den Displays der Visualisierungs-Ansicht geladenen Objekte austauschen, d.h. in die Displays andere Objekte anstelle der zuvor angezeigten Objekte laden, können Sie über die Standard-Funktionen des Internet Explorers Schritt für Schritt die jeweils zuletzt vorgenommene Änderung rückgängig machen, d.h wieder das zuvor angezeigte Objekt in das entsprechende Display laden.

Wenn Sie ATCM Console außerhalb von ATCM Engineering gestartet haben, benutzen Sie dazu die Schaltfläche Zurück auf der Werkzeugleiste des Internet Explorers. Mit jedem Klick auf Zurück nehmen Sie eine Änderung in der umgekehrten Reihenfolge, in der Sie diese vorgenommen haben, zurück. In jedem Display, in dem Sie ein neues Objekt geladen haben, wird also in der Reihenfolge der vorgenommenen Änderungen das zuvor geladene Display wieder angezeigt.
Analog dazu navigieren Sie mit der Schaltfläche Vorwärts auf der Werkzeugleiste des Internet Explorers wieder Schritt für Schritt diesmal in entgegengesetzter Richtung durch die in den Displays dargestellten Objekte. Änderungen in den dargestellten Displays, die Sie über Zurück zurückgenommen haben, werden damit wieder in Kraft gesetzt. Die Schaltfläche Vorwärts ist nur dann aktiv, wenn Sie über Zurück schon Änderungen vorgenommen haben.

Wenn Sie ATCM Console aus ATCM Engineering heraus gestartet haben, bieten Ihnen die Schaltflächen auf der Internet-Navigationsleiste von ATCM Engineering dieselbe Funktionalität wie die Schaltflächen Zurück und Vorwärts.
