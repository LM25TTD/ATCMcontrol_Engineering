Mit Layouts arbeiten

Layouts bilden die Basisstruktur für die Anordnung der in der Visualisierungsansicht dargestellten Displays. Die Anordnung von Displays kann über den Layout-Editor oder über die Kontext-Menüs der Displays in der Visualisierungs-Ansicht geändert werden.

Änderungen, die Sie an Layouts vornehmen, erfolgen immer auf der Basis des aktuell aktiven Layouts. Welches Layout aktiv ist , wird im Listenfeld Layout auf der Werkzeugleiste von ATCM Control angezeigt. Falls Sie noch kein neues Layout gespeichert haben, ist das zugrundeliegende Basis-Layout das von ATCM Control verwendete Standardlayout.

Mit dem Layout-Editor oder über das Kontext-Menü der Displays vorgenommene Änderungen des Layouts werden nicht automatisch gespeichert. Sobald Sie die gesamte Visualisierungsansicht über die entsprechende Option des Internet Explorers aktualisieren oder ATCM Console neu starten, werden die Displays wieder in der Anordnung des aktuell geladenen Layouts angezeigt. Um Änderungen an einem Layout dauerhaft beizubehalten, müssen Sie das Layout explizit speichern.
Zum Speichern und Laden eines Layouts siehe auch die Abschnitte: Geänderte Layouts speichern und Gespeicherte Layouts laden.

Jedesmal, wenn Sie ATCM Console starten, werden die Displays auf Basis des zuletzt verwendeten Layouts angeordnet. Wenn Sie noch kein neues Layout angelegt und gespeichert haben, wird für die Anordnung der Displays das Standardlayout von ATCM Console verwendet.

Ein Layout im Layout Editor editieren

Um ein Layout zu editieren, starten Sie den Layout Editor, indem Sie auf die Schaltfläche Edit auf der Werkzeugleiste von ATCM Console oder auf die Option Edit Layout im Kontext-Menü eines beliebigen Displays klicken.

Die im Layout Editor gezeigte Anordnung der Displays entspricht immer dem Layout, das aktuell in ATCM Console angezeigt wird, auch wenn dieses noch nicht gespeichert wurde. Wenn Sie noch kein Layout in ATCM Console bearbeitet haben, sehen Sie die in der obigen Abbildung gezeigte Struktur. Diese ist identisch mit dem in ATCM Console verwendeten Standardlayout. Wie Sie durch Vergleich beider Bilder erkennen können, entspricht jeweils einem Display eine Reihe von Quadraten mit gleicher Farbe und Nummer im Layout Editor. In obiger Abbildung wird also das erste Display von links von den grauen Quadraten mit der Nummer 1, das zweite Display von den blauen Quadraten mit der Nummer 2 usf. im Layout Editor repräsentiert.

Sie können Größe und Position der Displays im Layout Editor ändern. Dazu gehen Sie folgendermaßen vor:

  1. Ganz oben im Layout Editor befindet sich eine Reihe von numerierten farbigen Quadraten. Jedes Quadrat repräsentiert ein separates Display. Klicken Sie mit der linken Maustaste auf eines dieser Quadrate. Das angeklickte Quadrat wird jetzt ähnlich einer gedrückten Schaltfläche versunken dargestellt, und signalisiert damit, daß es aktiv ist.
  2. Der sich unterhalb dieser Quadrate befindende Bereich repräsentiert das in der Visualisierungs-Ansicht dargestellte Layout. Das Layout ist in 4 x 4 Quadrate eingeteilt. Diese bilden ein Raster, das mögliche Positionen für die Plazierung eines Displays angibt. Klicken Sie in eines der Quadrate. Das Quadrat wird jetzt in der Farbe und mit der Nummer dargestellt, die Sie zuvor durch Mausklick auf eines der Quadrate in der oberen Reihe aktiviert haben.
  3. Um die Fläche, die ein Display einnehmen soll, festzulegen, klicken Sie auf weitere Quadrate, so daß diese eine zusammenhängende Fläche bilden. Die Fläche, die ein Display später in der Visualisierungsansicht einnehmen wird, wird durch die Quadrate mit gleicher Farbe und Nummer bestimmt. Es ist auch möglich, ein Display, das aus nur einem einzigen Quadrat besteht, zu erzeugen.
  4. Wenn Ihr Display die gewünschte Position und Größe hat, klicken Sie wieder in der oberen Reihe der Quadrate auf ein anderes Quadrat, um Position und Größe eines weiteren Displays festzulegen. Alle Aktionen, die Sie ab jetzt in dem sich unterhalb der Quadratreihe befindenen Bereich vornehmen, werden jetzt mit der Farbe und Nummer des aktivierten Quadrats ausgeführt.
  5. Um den Layout Editor wieder zu schließen und das Ergebnis Ihrer Anordnung anzusehen, klicken Sie auf OK. Wollen Sie dagegen Ihre Änderungen am Layout verwerfen, klicken Sie auf Cancel.

Regeln zur Gestaltung gültiger Layouts

Displays können im Layout Editor nicht beliebig angeordnet werden. Sie müssen daher einige Regeln einhalten, um ein Layout zu erstellen, das als gültiges Layout in der Visualisierungsansicht angezeigt werden kann:

  • Quadrate derselben Farbe und Nummer müssen so angeordnet werden, daß sie einen zusammenhängenden Bereich bilden, da ein Display nicht auf mehrere räumlich nicht zusammenhängede Stellen in der Visualisierungs-Ansicht verteilt werden kann.
  • Displays dürfen nicht L-förmig sein, d.h Quadrate derselben Farbe und Nummer dürfen nicht in L-Form angeordnet werden.
  • Displays dürfen nicht kreisförmig angeordnet werden (siehe dazu Beispiel für kreisförmige Anordnung).

Ob die aktuelle Anordnung von Displays ein gültiges Layout ergibt, wird im Layout-Editor in der Textzeile links neben den beiden Schaltflächen OK und Cancel angezeigt. Ergibt die aktuelle Anordnung der Displays ein gültiges Layout, zeigt die Textzeile den Text Faceplate Layout: Valid an. Ergibt die aktuelle Anordnung kein gültiges Layout, erscheint dagegen der Text Faceplate Layout: Invalid. Die Schaltfläche OK ist in diesem Fall deaktiviert, so daß ein ungültiges Layout nicht in die Visualisierungs-Ansicht übernommen werden kann.

Beispiele für ungültige und gültige Layouts finden Sie im Abschnitt Beispiele für Display-Layouts.

Layouts über das Kontext-Menü der Displays bearbeiten

Jedes Display verfügt über ein Kontext-Menü, über das sich Position und Größe des Displays beeinflussen lassen. Über Expand-Optionen können Displays nach links oder rechts expandiert , über Insert-Optionen können weitere Displays in die Visualisierungs-Ansicht eingefügt werden. Über das Kontext-Menü vorgenommene Änderungen werden in das aktuell aktive Layout übernommen und im Layout-Editor angezeigt.
Zusätzlich enthält das Kontext-Menü zwei Optionen zum Zurücksetzen des im Display dargestellten Objekts und die Optionen Edit Layout zum Bearbeiten und Copy Layout zum Kopieren eines Layouts (zu diesen Optionen siehe die Abschnitte Ein Layout im Layout Editor bearbeiten und Ein geändertes Layout speichern).

Die Optionen des Kontext-Menüs sind kontextsensitiv, d.h. es werden nur die Optionen im Menü angezeigt, die im Zusammenhang mit der aktuellen Position eines Displays sinnvoll sind. Befindet sich ein Display bspw. bereits am linken Ende der Visualisierungs-Ansicht, kann dieses Display nicht weiter nach links expandiert werden. Die Option Expand Left erscheint in diesem Fall also nicht im Kontext-Menü des entsprechenden Displays. Generell gilt, daß ein Display nur in die Richtung expandiert werden kann, in der sich noch andere Displays befinden. Analog gilt für das Einfügen von Displays über die Insert-Optionen, daß die Insert-Optionen zum Einfügen von Displays nur dann angezeigt werden, wenn die maximale Anzahl von 4 Displays in einer vertikalen oder horizontalen Reihe der Visualisierungs-Ansicht noch nicht erreicht ist.

Beim Einfügen eines zusätzlichen Displays über die Insert-Optionen wird in der Regel ein leeres Display eingefügt. Haben Sie allerdings vor dem Einfügen eines Displays ein oder mehrere Displays über eine Expand-Option aus der Visualisierungs-Ansicht entfernt (siehe Abschnitt Ein Display nach links erweitern), wird in den eingefügten Displays das zuerst entfernte und bis zum Zeitpunkt des Einfügens nicht mehr angezeigte Objekt geladen.

Sie rufen das Kontext-Menü eines Displays auf, indem Sie mit der rechten Maustaste auf die Kopfleiste des gewünschten Displays klicken. Beachten Sie, daß nur bei Klick auf die Display-Kopfleiste das displayspezifische Kontext-Menü aufgerufen wird, während Sie bei Klick auf den Darstellungsbereich des Displays ein Kontext-Menü öffnen, das Optionen des Internet Explorers enthält.

Ein Display nach links erweitern

Über die Option Expand Left erweitern Sie ein Display nach links. Das sich vor dem Aktivieren dieser Option auf der linken Seite des selektierten Displays befindende Display wird nicht mehr angezeigt. Statt dessen nimmt nun das expandierte Display zusätzlich zu der Fläche, die es zu vor belegt hat, auch die Fläche des Displays ein, das sich zuvor auf dessen linker Seite befunden hat.

Expandieren Sie bspw. das unten im Bild zu sehende Display, das die Funktionsbaustein-Instanz time_count visualisiert, wird das die Funktionsbaustein-Instanz pulse darstellende Display durch Fläche des nach links erweiterten Displays ersetzt.

Ein Display nach rechts erweitern

Über die Option Expand Right erweitern Sie ein Display nach rechts. Die Anzeige eines nach rechts erweiterten Displays erfolgt analog zur Erweiterung eines Displays nach links.

Ein Display nach oben erweitern

Falls sich über einem Display ein weiteres Display befindet, können Sie das unten liegenden Display über die Option Expand Up nach oben erweitern. Das sich vor dem Aktivieren dieser Option oberhalb des selektierten Displays befindende Display wird nicht mehr angezeigt. Statt dessen nimmt nun das expandierte Display zusätzlich zu dem Raum, den es zu vor belegt hat, auch den Raum des Displays ein, das sich zuvor oberhalb des expandierten Displays befunden hat.

Expandieren Sie bspw. das unten im Bild zu sehende Display, das die Funktionsbaustein-Instanz time-count visualisiert, wird das die Funktionsbaustein-Instanz rise_edge2 darstellende Display durch die Fläche des nach oben erweiterten Displays ersetzt.

Ein Display nach unten erweitern

Falls sich unterhalb eines Displays ein weiteres Display befindet, können Sie das oben liegende Display über die Option Expand Down nach unten erweitern. Die Anzeige eines nach unten erweiterten Displays erfolgt analog zur Erweiterung eines Displays nach oben.

Ein Display auf der linken Seite einfügen

Über die Option Insert Left können Sie ein Display auf der linken Seite eines selektierten Displays einfügen. Das selektierte Display wird um die Fläche verkleinert, die das auf der linken Seite eingefügte Display zur Darstellung benötigt.

Fügen Sie bspw. links nebem dem unten im Bild zu sehenden Display, das die Funktionsbaustein-Instanz timer_base visualisiert, ein Display ein, wird das Display um die Fläche verkleinert, die das neu eingefügte Display zur Darstellung benötigt.

Ein Display oberhalb eines selektierten Displays einfügen

Fügen Sie bspw. über dem unten im Bild zu sehenden Display, das die Funktionsbaustein-Instanz time_count visualisiert, ein Display ein, wird das Display um die Fläche verkleinert, die das neu eingefügte Display zur Darstellung benötigt.

Display-Inhalt zurücksetzen

Über die Option Clear des Kontext-Menüs setzen Sie das Display in einen leeren Zustand zurück. Das zuvor in dem ausgewählten Display dargestellte Objekt wird nicht mehr angezeigt und die Kopfleiste des Displays zeigt die Beschriftung Empty.

Über die Option Clear All wird der Inhalt aller in ATCM Console angezeigten Displays in eine leeren Zustand zurückgesetzt.

Geänderte Layouts speichern

Änderungen, die Sie an einem Layout vorgenommen haben, werden nicht automatisch gespeichert und gehen daher verloren, sobald Sie ATCM Console schließen oder die gesamte Visualisierungs-Ansicht aktualisieren. Sie müssen daher ein Layout explizit speichern, um Änderungen an einem Layout dauerhaft beizubehalten.

Grundlage für das Speichern eines Layouts ist die aktuell in der Visualisierungs-Ansicht dargestellte Anordnung der Displays. Gespeichert werden sowohl Änderungen, die über den Layout-Editor, als auch Änderungen, die über das Kontext-Menü vorgenommen wurden. Haben Sie an einem Layout Änderungen vorgenommen und noch nicht gespeichert, wird im Listenfeld Layout auf der Werkzeugleiste von ATCM Console der Eintrag [Unsaved Layout] angezeigt.
Beim Speichern eines Layouts wird nicht nur die aktuelle Position und Größe der Displays, sondern auch die Verknüpfung zu den Objekten, die aktuell in den Displays dargestellt werden, gespeichert. Wenn Sie ein gespeichertes Display laden, werden daher auch die den Displays zugeordneten Objekte wieder in den entsprechenden Displays visualisiert, sofern noch eine gültige Verbindung zu dem betreffenden Objekt besteht. Besteht die Verbindung zu dem Control, auf dem das Objekt geladen wurde, nicht mehr, oder wurde das Objekt aus dem Projekt entfernt, wird in den den entsprechenden Objekten zugeordneten Displays die Meldung The object <object name> has been removed ! angezeigt.

Gehen sie folgendermaßen vor, um ein geändertes Layout zu speichern:

  1. Klicken Sie nach dem Editieren eines Layouts auf die Schaltfläche Disk auf der Werkzeugleiste von ATCM Engineering,

    - ODER -

    klicken Sie mit der rechten Maustaste auf die farbige Kopfleiste eines beliebigen Displays, um das Kontext-Menü aufzurufen, und wählen Sie dort die Option Copy Layout.

  2. Geben Sie den Namen des neuen Layouts im unteren Eingabefeld des Dialogs ein. Die Schaltfläche Save as wird mit der Eingabe des ersten Zeichens aktiviert.



  3. Klicken Sie auf die Schaltfläche Save as, um das Layout in die Liste Available Layouts zu übernehmen. Das Layout steht ab jetzt unter dem neuen Namen im Listenfeld der Werkzeugleiste von ATCM Console zur Auswahl zur Verfügung.

  4. Klicken Sie auf Exit, um das Dialogfeld Save Layout wieder zu schließen. Das gespeicherte Layout wird automatisch zum aktuell aktiven Layout.

Sie können mehrere Layouts speichern, indem Sie nach dem Ändern eines Layouts wieder den Dialog Save and Delete Layout aufrufen und einen neuen Namen für das geänderte Layout eingeben. Das Layout wird wie zuvor wieder in die Liste Available Layouts übernommen.
Nach dem Speichern wird das zuletzt gespeicherte Layout automatisch zum aktuell aktiven Layout und im Listenfeld Layout auf der Werkzeugleiste von ATCM Console angezeigt.

Gespeicherte Layouts werden in ATCM Console intern verwaltet und nicht als Dateien abgelegt. Das Kopieren von Layouts auf Diskette zum Einsatz auf einem anderen Rechner ist daher nicht möglich.

Ein gespeichertes Layout laden

Wenn Sie bereits ein oder mehrere Layouts gespeichert haben, wird das momentan geladene Layout im Listenfeld Layout auf der Werkzeugleiste von ATCM Console angezeigt.

Um ein anderes Layout zu laden

  1. klicken Sie im Listenfeld Layout auf die Schaltfläche mit dem nach unten zeigenden Pfeil, um die Liste des Listenfelds aufzuklappen. In der Liste werden alle gespeicherten Layouts angezeigt.
  2. Klicken Sie mit der linken Maustaste auf das gewünschte Layout, um dieses zu laden. Nach einem kurzen Zeitraum, den ATCMControl für den Neuaufbau der Visualisierungsansicht benötigt, werden die Displays im selektierten Layout angezeigt.

Ein Layout löschen

  1. Klicken Sie auf die Schaltfläche Disk, um den Dialog Save Layout aufzurufen.

  2. In der Liste Available Layouts werden alle gespeicherten Layouts aufgeführt. Wählen Sie in der Liste das gewünschte Layout aus. Sobald Sie ein Layout markiert haben, wird die zuvor inaktive Schaltfläche Delete aktiviert, und der Name des selektierten Layouts im unteren Feld des Dialogs angezeigt. Wenn Sie während der Auswahl der Layouts die STRG- oder die Shift-Taste gedrückt halten, können Sie mehrere Layouts auf einmal auswählen.



  3. Klicken Sie auf Delete, um das selektierte Layout zu löschen.

  4. Klicken Sie auf Exit, um den Dialog zu schließen.
