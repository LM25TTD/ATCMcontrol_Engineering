Verbindungsmöglichkeiten / Starten von ATCM Console

Eine Konfiguration kann in ATCM Engineering nicht nur dem lokalen Rechner, sondern auch einem Remote-Rechner zugewiesen werden. Konfiguration und PC sind in ATCMControl nach der Zuweisung einer Konfiguration an einen PC praktisch identisch. Eine Konfiguration wird zur Ausführung auf das Control des Rechners geladen, der der entsprechenden Konfiguration zugeordnet wurde. Es ist auf diese Weise bspw. möglich, ein Projekt in ATCM Engineering auf dem lokalen Rechner zu compilieren, und auf einem Remote-Rechner auszuführen, d.h. auf das Control, das sich auf dem Remote-Rechner befindet, zu laden. Falls Sie über eine Lizenz für Mehrfachkonfigurationen verfügen, können Sie in einem Projekt auch mehrere Konfigurationen anlegen, die jeweils einem anderen PC zugewiesen sind und daher auf dem Control des jeweiligen PCs ausgeführt werden.

In einer gestarteten Instanz von ATCM Console werden jeweils die Prozeßdaten einer einzelnen Konfiguration visualisiert. ATCM Console stellt dazu über die OPC-Schnittstelle eine Verbindung zu dem Control des der Konfiguration zugewiesenen Rechners her. Vom lokalen Rechner aus kann sowohl die Konfiguration, die auf dem Control des lokalen Rechners ausgeführt wird, als auch eine Konfiguration, die auf einem Remote-Rechner läuft, in ATCM Console visualisiert und beobachtet werden. Ebenso ist es möglich, dieselbe auf ein ATCM Control geladene Konfiguration von mehreren Remote-Rechnern aus in ATCM Console zu beobachten.

Die Auswahl der zu visualisierenden Konfiguration und damit des entsprechenden Controls erfolgt bereits mit dem Start von ATCM Console, d.h. der Kontext, aus dem heraus ATCM Console gestartet wurde, entscheidet darüber, von welchem Control die visualisierten Daten stammen. Zum Starten von ATCM Console stehen drei Möglichkeiten zur Verfügung:

  • Starten über das ATCM Tray auf der Windows Programmleiste
  • Starten aus ATCM Engineering
  • Starten über die Eingabe der Startadresse im Adressfeld des Internet Explorers

Bei mehreren Konfigurationen in einem einzigen Projekt kann die Konfiguration auch noch nach dem Start von ATCM Console gewechselt werden.

Beim Start von ATCM Console über das ATCM Tray oder durch Eingabe einer Startadresse im Adressfeld des Internet Explorers können alle im Projekt enthaltenen Objekte über den Navigator von ATCM Console in ein Display geladen und dort angezeigt werden. Die Anordnung der diesen Objekten zugeordneten Displays kann über das Kontext-Menü oder den Layout-Editor verändert werden (siehe Abschnitt Mit Layouts arbeiten). Wird ATCM Console dagegen aus ATCM Engineering gestartet, ist die Auswahl der Objekte, die in ein Display geladen werden können, abhängig von der Aufrufposition von ATCM Console (siehe Starten von ATCM Console aus ATCM Engineering).

Unabhängig davon, auf welche Weise ATCM Console gestartet wurde, können mehrere Instanzen von ATCM Console zugleich aufgerufen werden. Um Probleme beim Arbeiten mit mehreren Instanzen von ATCM Console zu vermeiden, sollten Sie in den in den Internetoptionen des Internet Explorers in der Registerkarte Erweitert das Kontrollkästchen Bowserfenster in einem eigenen Prozeß öffnen markieren.

Starten von ATCM Console über das ATCM Tray

Beim Start über das ATCM Tray wird ATCM Console als Standalone-Applikation innerhalb des Internet-Explorers geöffnet. In einer über das ATCM Tray gestarteten ATCM Console werden immer die Prozeßdaten des sich auf dem lokalen Rechner befindenden Controls visualisiert.

Um ATCM Console über das ATCM Tray zu starten

klicken sie mit der linken Maustaste doppelt auf das sich am rechten bzw. unteren Ende der Windows-Taskleiste befindende ATCMControl Symbol

- ODER -

klicken Sie mit der rechten Maustaste auf das ATCMControl-Symbol , um dessen Kontext-Menü zu öffnen, und wählen Sie Open ATCM Console.

Starten von ATCM Console aus ATCM Engineering

Beim Start aus ATCM Engineering wird ATCM Console als Fenster innerhalb von ATCM Engineering geöffnet, d.h. das ATCM Console-Fenster kann nur innerhalb von ATCM Engineering verschoben werden. Es können mehrere Instanzen von ATCM Console aus ATCM Engineering aufgerufen werden. Jede Instanz wird in einem eigenen Fenster innerhalb von ATCM Engineering dargestellt.

Ein aus ATCM Engineering gestartetes Konsolenfenster enthält mit Ausnahme des Kontext-Menüs nur die ATCM Control-spezifischen Bedien- und Visualisierungselemente. Ein Teil der Internet Explorer-Funktionalität wird durch eine Schaltflächengruppe ersetzt, die die wichtigsten Navigationsfunktionen zur Verfügung stellt. Erläuterungen zu dieser Schaltflächengruppe finden Sie im Abschnitt Internet-Navigationsleiste in ATCM Engineering.

Der Start von ATCM Console aus ATCM Engineering erfolgt über das Kontext-Menü des Navigators von ATCM Engineering.
Das Kontext-Menü des Engineering-Navigators mit einem Menüpunkt zum Öffnen von ATCM Console kann von folgenden Positionen aus aufgerufen werden:

In der Hardware-Ansicht
von einem PC-Symbol
In der Instance-Ansicht
von dem Symbol einer Konfiguration ,
einer Ressource
oder einer Task.

Klicken Sie nach der Auswahl eines dieser Symbole mit der rechten Maustaste, um das Kontext-Menü aufzurufen und wählen Sie dort die Option Open ATCM Console.

Mit welchem Control ATCM Console beim Start aus ATCM Engineering verbunden und welches Objekt der Konfiguration visualisiert wird, ist abhängig von der Aufrufposition, die für das Öffnen des Kontext-Menüs gewählt wurde.

Falls im aktuell in ATCM Engineering geöffneten Projekt nur eine Konfiguration angelegt wurde, werden immer die Prozeßdaten visualisiert, die auf dem Control geladen sind, das sich auf dem der Konfiguration zugewiesenen PC befindet. Dies kann sowohl der lokale PC, als auch ein Remote-Rechner sein. Die gesamten in der Konfiguration enthaltenenen Objekte stehen im Navigator von ATCM Console zur Auswahl zur Verfügung, wenn die Aufrufposition des Kontext-Menüs das PC-Symbol oder das Konfigurations-Symbol ist. Beim Aufruf des Kontext-Menüs aus der Position eines Ressourcen-Symbols wird die Taskliste der Konfiguration angezeigt, beim Aufruf aus der Position eines Task-Symbols dagegen das Faceplate der zuvor selektierten Task (Näheres zu Navigator und Faceplates in ATCM Console erfahren Sie in den Abschnitten Benutzeroberfläche und Displays). Der Navigator wird bei Aufruf aus den letzten beiden Positionen nicht eingeschränkt; die Auswahl der Objekte ist daher eingeschränkt.

Falls Sie eine Lizenz für Mehrfachkonfigurationen besitzen, können Sie in einem Projekt auch mehrere Konfigurationen angelegen. In diesem Fall wird ATCM Console mit dem Control verbunden, das sich auf dem PC befindet, dessen zugeordnete Konfiguration oder ein sich unter der Konfiguration befindendes Symbol als Aufrufposition für das Kontext-Menü ausgewählt wurde. Beim Start aus der im folgende Bild zu sehenden Aufrufposition des Kontext-Menüs wird ATCM Console bspw. mit dem Configuration2 zugeordneten Control verbunden.

Starten von ATCM Console durch Eingabe der Startadresse im Internet Explorer

Beim Start von ATCM Console aus dem Internet Explorer kann bei Eingabe der Startadresse der PC angegeben werden, mit dessen Control ATCM Console verbunden werden soll:

  1. Starten Sie den Internet Explorer.
  2. Geben Sie im Feld Adresse die gewünschte Startseite ein. Die Adresse muß folgender Syntax entsprechen:
    'http//<Name des PC's>/4C'. Hat Ihr PC bspw. den Namen Automation1, lautet die Adresse 'http//Automation1/4C'.
  3. Klicken Sie auf Wechseln zu oder drücken Sie die Enter-Taste, um die eingegebene Adresse aufzurufen.

Paßwortschutz

In ATCM Engineering kann ein Paßwort für das aktuell geladenene Projekt vergeben werden (siehe auch den Abschnitt projektbezogenes Paßwort vergeben), das nach der Compilation des Projekts wirksam wird. Wenn eine in einem paßwortgeschützten Projekt enthaltenen Konfiguration auf ein ATCM Control geladen wird, erfolgt beim Start von ATCM Console eine Paßwortabfrage. Die Eingabe eines Paßworts wird bei paßwortgeschützten Projekten unabhängig davon verlangt, ob das zu verbindende Control sich auf dem lokalen oder auf einem Remote-Rechner befindet. Die Abfrage erfolgt allerdings nur, wenn ATCM Console außerhalb von ATCM Engineering, also über das ATCM Tray oder über das Adressfeld des Internet Explorers gestartet wurde.

Die Sicherheitsüberprüfung unterscheidet zwei Zugriffsstufen auf die Konsole:

  • Nur Lese-Zugriff: Die Konsole kann gestartet, und die Prozeß-Variablen können überwacht werden. Die Einschränkung des Nur- Lese-Zugriffs besteht darin, daß normalerweise veränderbare Werte von Prozeß-Variablen nicht verändert werden können. Ebensowenig können Tasks und Resourcen gestartet oder gestoppt werden.
  • Uneingeschränkter Zugriff: Werte von Prozeß-Variablen können gelesen als auch verändert werden, falls ein Schreibzugriff auf die entsprechende Variable vom zugeordneten Display vorgesehen ist. Tasks und Resourcen können gestartet und gestoppt werden.

Welche Zugriffsstufe für eine Konfiguration gültig ist, wird bei Vergabe des Paßworts in ATCM Engineering (im Menü Tools - Project Settings... Registerkarte Console) festgelegt.

Beachten Sie bitte, daß die Paßwortabfrage auch dann erfolgt, wenn Sie bspw. nach einem Download die gesamte Visualisierungs-Ansicht von ATCM Console über die Schaltfläche Aktualisieren auf der Werkzeugleiste des Internet Explorers oder ein einzelnes Display über das Explorer Kontext-Menü des Displays aktualisieren. In letzterem Fall wird der Paßwort-Dialog nur in das aktualisierte Display eingeblendet, da die restlichen Displays nicht von der Aktualisierung betroffen sind.

Einloggen in ATCM Console

Um sich im Falle eines Paßwortschutzes in ATCM Console einzuloggen, geben Sie das Paßwort in das Eingabefeld ein und bestätigen Ihre Eingabe durch Klicken auf Enter.
Falls Sie sich verschrieben haben, können Sie das Eingabefeld durch Klicken auf Reset wieder in seinen leeren Ausgangszustand zurücksetzen, und die Eingabe wiederholen.

Wenn Sie ein ungültiges Paßwort eingeben, wird der Zugriff auf die Konsole verweigert und das Login-Dialogfeld neu gestartet.
Wenn Sie dagegen ein gültiges Paßwort eingeben, wird eine Konsolen-Sitzung einschließlich aller Zugangsrechte, die über das eingegebene Paßwort zur Verfügung gestellt werden, gestartet.

Eine Verbindung zu ATCM Control beenden

Eine bestehende Verbindung wird abgebrochen, wenn die folgenden Ereignisse auftreten:

  • Die Logout-Schaltfläche in der Konsole wird gedrückt.
  • Auf das mit ATCM Console verbundene Control wird ein neues Projekt geladen.
  • Die Verbindung wird bspw. durch Schließen des Internet Explorers beendet und nicht in einem Zeitraum von c.a. 30 Sekunden erneut gestartet. In diesem Fall gehen die auf dem Server abgelegten Daten zur aktuellen Verbindung verloren.

Die Logout-Schaltfläche befindet sich auf der Symbolleiste von ATCM Console. Sie wird nur dann angezeigt, wenn die geladenene Konfiguration mit einem Paßwort geschützt ist. Besitzt die aktuell geladenen Konfiguratio kein Paßwort, ist diese Schaltfläche dagegen nicht sichtbar.
Bei Klick auf die Logout-Schaltfläche wird die bestehende Verbindung beendet, und der Login-Dialog wieder angezeigt.
Wie Sie aus dem Feld Adresse im Internet Explorer oder - falls Sie ATCM Console aus ATCM Engineering gestartet haben - sehen können, bezieht sich der Paßwort-Dialog immer noch auf den Zugriff auf das Control des Rechners, mit dem Sie zuvor eine Verbindung hergestellt haben. Wenn Sie sich nach dem Ausloggen erneut mit demselben Paßwort wieder einloggen, stellen Sie wieder eine Verbindung zu dem zuvor verbundenen Control her.
