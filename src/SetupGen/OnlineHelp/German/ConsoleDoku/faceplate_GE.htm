Einführung

Displays sind die Visualisierungseinheiten innerhalb der Visualisierungs-Ansicht, in denen jeweils ein Objekt des in ATCM Console beobachteten Projekts geladen werden kann. Jeder Objektart (z.B. Task, Funktionsbaustein-Instanz) sind bestimmte Visualisierungschemata zugeordnet, die regeln, welche Daten im Display zur Darstellung kommen (einen Überblick über die in der Visualisierungs-Ansicht darstellbaren Objekte erhalten Sie im Abschnitt Die Registerkarte Project), und die dafür sorgen, daß Objekte, die der gleichen Objektart angehören, immer auf die gleiche Weise dargestellt werden.

Für eine große Anzahl von Objekten stehen zwei Visualisierungsschemata zur Verfügung:
das Tuning Display und das Faceplate:

  • Das Tuning Display dient der Anzeige von Werten des in ein Display geladenen Objekts. Diese Werte werden im Tuning Display in Feldern angezeigt. In Instanzen von Funktionsbausteinen und Strukturen können diese Werte zum Teil zur Laufzeit des Control verändert werden.
    Siehe auch die Abschnitte Instanzen von Funktionsbausteinen und Instanzen von Strukturen.
  • Wie das Tuning Display zeigt auch das Faceplate Werte eines in ein Display geladenen Objekts an. Gegenüber dem Tuning Display bietet das Faceplate eine weitere Palette von Visualisierungselementen. Darüberhinaus kann auch der Laufzeitstatus einiger Objekte über Schaltflächen beeinflußt werden.

Standardmäßig steht für die Visualisierung von Funktionsbaustein-Instanzen nur ein Tuning Display zur Verfügung Einige erweiterte Funktionsbausteine der IEC-Standard- und der 4C_Application-Bibliothek verfügen jedoch für die Visualisierung zusätzlich zum Tuning Display über ein Faceplate mit zusätzlichen Elementen zur grafischen Darstellung von Werten.

Benutzerdefinierte Displays können analog zu den Displays der erweiterten Funktionsbausteine erzeugt werden. Voraussetzung dafür sind allerdings vertiefte Kenntnisse in HTML, Java, Java Script, ASP und der OPC-Schnittstelle.

SFC-Bausteinen, die mit Hilfe des Grafik-Editors programmiert wurden verfügen über zusätzliche Visualierungselemente. Näheres hierzu siehe SFC-Bausteine in ATCM Console.

Standardmäßig wird beim Laden eines Objekts in ein Display zuerst das Faceplate (sofern für das Objekt implementiert) angezeigt. Um von der Darstellung des Objekts im Faceplate zur Darstellung im Tuning Display zu wechseln, steht Ihnen am rechten Ende der Kopfleiste des Displays ein Link mit der Beschriftung TD zur Verfügung. Nach dem Wechsel zum Tuning Display enthält die Kopfleiste des Displays einen Link mit der Beschriftung FP, über den Sie wieder zum Faceplate wechseln können.

Die Configuration List

Die Configuration List zeigt in jeder Zeile die Daten einer auf ein ATCM Control geladenen Ressource an. Alle auf ein Control geladenen Ressourcen, die zum aktuell beobachteten Projekt gehören, werden hier aufgeführt. In den Spalten werden folgende Informationen zu jeder Ressource ausgegeben:

Status
Zeigt den Laufzeitstatus einer Ressource an. Es werden 4 Laufzeitzustände unterschieden:
  • Grüne Farbe und Meldung Running: Das Control, auf das die Ressource geladen wurde, läuft.
  • Gelbe Farbe und Meldung Stopped: Das Control, auf das die Ressource geladen wurde, wurde gestoppt.
  • Gelbe Farbe und Meldung Loading: Die Ressource wird auf ein Control geladen.
  • Rote Farbe und Meldung Error: Bei der Ausführung der Ressource auf dem Control ist ein Fehler aufgetreten.
Machine
Zeigt den PC an, dem die Ressource zugeordnet wurde.
Resource
Zeigt den Namen der Ressource an. Der Ressource-Name wird in blauer Farbe und unterstrichen dargestellt und signalisiert damit, daß es sich bei dem Namen um einen Hyperlink handelt. Wenn Sie auf den Namen einer hier aufgeführten Ressource klicken, wird statt der Configuration List die angeklickte Ressource in das Display geladen.
Memory (kB)
Zeigt die Speichermenge in Kilobyte an, die die Ressource auf dem Control benötigt.
Project
Zeigt den Namen des Projekts an, zu dem die Ressource gehört.

Die Ressource

Das Display für eine Ressource zeigt Daten zum Laufzeitstatus des zugeordneten Controls an. Über die zwei Schaltflächen im Faceplate kann das Control, auf dem die Ressource geladen wurde, gestartet und gestoppt werden.

Faceplate

Laufzeitstatus
Die vier farbigen Quadrate zeigen Laufzeitstatus der Ressource an. Es werden 4 Laufzeitzustände unterschieden:
  • Grüne Farbe und Meldung Running: Das Control, auf das die Ressource geladen wurde, läuft.
  • Gelbe Farbe und Meldung Stopped: Das Control, auf das die Ressource geladen wurde, wurde gestoppt.
  • Gelbe Farbe und Meldung Loading: Die Ressource wird auf ein Control geladen.
  • Rote Farbe und Meldung Error: Bei der Ausführung der Ressource auf dem Control ist ein Fehler aufgetreten.
Memory (kB)
Zeigt die Speichermenge in Kilobyte an, die die Ressource auf dem Control benötigt.
Project
Zeigt den Namen des Projekts an, zu dem die Ressource gehört.
Diese Schaltfläche stoppt das der Ressource zugeordnete Control. Die Schaltfläche ist deaktiviert, wenn das Control bereits gestoppt ist.
Diese Schaltfläche startet das der Ressource zugeordnete Control. Die Schaltlfläche ist deaktiviert, wenn das Control läuft.
Configuration List
Bei Klick auf diesen Link wechselt das Display zur Configuration List.

Tuning Display

__state
Zeigt den Laufzeitstatus des Controls an, auf das die Ressource geladen wurde.
__memused
Zeigt die Speichermenge in Byte an, die die Ressource auf dem Control benötigt.
__projectname
Zeigt den Namen des Projekts an, zu dem die Ressource gehört.

Die Task List

Die Task List listet alle unter einer Ressource enthaltenen Tasks auf und zeigt deren Laufzeitstatus, Zykluszeit und Priorität an. Alle in der Task List aufgeführten Tasks können über Schaltflächen gestartet und gestoppt werden.

Status
Zeigt den Laufzeitstatus einer Task an. Es werden 4 Laufzeitzustände unterschieden:
  • Grüne Farbe und Meldung Running: Das Control, auf das die Ressource geladen wurde, läuft.
  • Gelbe Farbe und Meldung Stopped: Das Control, auf das die Ressource geladen wurde, wurde gestoppt.
  • Gelbe Farbe und Meldung Loading: Die Ressource wird auf ein Control geladen.
  • Rote Farbe und Meldung Error: Bei der Ausführung der Ressource auf dem Control ist ein Fehler aufgetreten.
Name
Name der Task. Bei Mausklick auf den Tasknamen wechselt das Display zu dem Faceplate der angeklickten Task.
Cycle Time(ms)
Zeigt die der Task zugeordnete Zykluszeit in Millisekunden an.
Priority
Zeigt die der Task zugeordnete Taskpriorität an.
Diese Schaltfläche stoppt alle in der Task List aufgeführten. Die Schaltfläche ist deaktiviert, wenn alle Tasks gestoppt sind.
Diese Schaltfläche startet alle in der Tasklist aufgeführten Tasks. Die Schaltlfläche ist deaktiviert, wenn alle Tasks laufen.

Die Message History

In der Message History werden alle in der Message-Ansicht von ATCM Engineering ausgegebenen Meldungen angezeigt und gesammelt. Zusätzlich zu den in ATCM Engineering ausgegebenen Meldungen erscheint auf der linken Seite des jeweiligen Meldungstextes die Nummer der Meldung und Datum und Uhrzeit, zu der die Meldung ausgegeben wurde. Zählung der Meldungsnummern beginnt bei 0, sodaß bspw. die 20. Meldung die Nummer 19 hat. Fehlermeldungen und Warnungen werden farblich hervorgehoben.

Die Anzeige von Meldungen in der Message-Ansicht erfolgt seitenweise. Ein Seite umfaßt jeweils 100 Meldungseinträge, die zusammen in der Message-Ansicht angezeigt werden. Wenn Sie mit Hilfe der vertikalen Scrolleiste durch die Meldungen scrollen, umfaßt der scrollbare Bereich also genau 100 Meldungen (Ausnahme siehe unten). Wollen Sie anderen Meldungen als die der aktuell geladenen Seite anzeigen, müssen Sie zu diesen Meldungen mit Hilfe der Schaltfächen unterhalb des Anzeigebereichs der Message-Ansicht navigieren. Die Schaltflächen haben folgende Funktionen:

Lädt die erste Seite. Es werden also die ersten 100 Meldungen angezeigt.
Lädt die vorhergehende Seite. Es wird um 100 Meldungen zurück navigiert.
Lädt die nächste Seite. Es wird um 100 Meldungen vorwärts navigiert (Ausnahme siehe unten).
Lädt die letzte Seite. Es werden die letzten 100 Meldungen angezeigt.
Zeigt die Meldung mit der im Feld rechts neben dieser Schaltfläche eingegebenen Nummer an.

Die Anzeige der Meldungen in der Message History wird nicht automatisch aktualisiert. Um auch die aktuellen Meldungen zu sehen, müssen Sie diese über die Schaltfläche zur Ansteuerung der letzten Seite einblenden. Falls seit der letzten Aktualisierung weniger als 100 Meldungen ausgegeben wurden, können Sie die Anzeige in der Message History auch über die Schaltfläche zur Navigation zu nächsten Seite aktualisieren. In diesem Fall beträgt die Anzahl der auf einer Seite enthaltenen Meldungen nur die Differenz zwischen der Nummer der letzten ausgegebenen Meldung und der letzten auf der Vorgängerseite ausgegebenen Meldung. Hatte bspw. die letzte auf der Ausgangsseite angezeigte Meldung die Nummer 1500 während die letzte vom Control ausgegebene Meldung die Nummer 1518 hat, enthält die Folgeseite nur 18 Meldungen.

Ausgegebene Meldungen werden von ATCM Control in einem Ringpuffer gespeichert. Dieser Puffer kann bis zu 100000 Meldungen speichern. Wird diese Anzahl überschritten, werden die zuerst ausgegebenen Meldungen aus dem Puffer entfernt, um Platz für die aktuellsten Meldungen im Puffer zu schaffen.
In der Kopfleiste der Message History werden die Anzahl der auf der aktuell geladenen Seite enthaltenen Meldungen und die Gesamtanzahl der im Ringpuffer gespeicherten Meldungen angezeigt. Beachten Sie bitte, daß die vom Control ausgegebenen Meldungen weitergezählt werden, auch wenn die maximale Anzahl von 100000 Meldungen bereits überschritten wurde. In diesem Fall wird in der Kopfleiste der Message History bspw. 100 of 99999 ausgegeben, während die aktuellste Meldung bereits die Nummer 120000 hat.

Tasks

Die Task-Displays visualisieten Werte zu Laufzeitzustand und Laufzeiteigenschaften der selektierten Task.

Faceplate

Laufzeitstatus
Die fünf farbigen Quadrate zeigen den Laufzeitstatus der Task an. Es werden 5 Laufzeitzustände unterschieden:
  • Grüne Farbe und Meldung Running: Die Task läuft.
  • Gelbe Farbe und Meldung Stopped: Die Task wurde gestoppt
  • Gelbe Farbe und Meldung Break: Die Task wurde an einem Breakpoint angehalten.
  • Gelbe Farbe und Meldung Loading: Die Task wird geladen.
  • Rote Farbe und Meldung Error: Bei der Task ist ein Fehler aufgetreten.
Cycle Time
Zeigt die der Task zugeordnete Zykluszeit an.
Elapsed Time
Zeigt die Zeit in Millisekunden an, die die Task für die Ausführung eines Zyklus benötigt.
Priority
Zeigt die der Task zugeordnete Taskpriorität an.
Diese Schaltfläche die Ausführung der Task. Die Schaltfläche ist deaktiviert, wenn Task bereits gestoppt ist.
Diese Schaltfläche startet eine Task. Die Schaltlfläche ist deaktiviert, wenn die Taskläuft.
Task List
Bei Klick auf diesen Link wechselt das Display zur Task List.

Tuning Display

__state
Zeigt den Laufzeitstatus der Task an.
__prio
Zeigt die der Task zugeordnete Taskpriorität an.
__cyle
Zeigt die der Task zugeordnete Zykluszeit an.
__etmax
Zeigt die Zeit in Millisekunden an, die die Task für die Ausführung eines Zyklus benötigt.
__errno
Zeigt den Fehlerstatus der Task an. Eine Fehlernummer ungleich 0 signalisiert einen Fehler.

Instanzen von Funktionsbausteinen

Funktionsbausteininstanzen werden, sofern in einem benutzerdefinierten Funktionsbaustein keine zusätzlichen Visualisierungsfunktionen implementiert wurden, nur als Tuning Display dargestellt. Die Kopfleiste des Displays enthält deshalb keinen Link zu einem entsprechenden Faceplate.

Im Display werden nur Daten vom Typ ANY_SIMPLE visualisiert, die zu der öffentlichen Schnittstelle des Funktionsbaustein gehören, also Variablen, die in einem VAR_INPUT- , VAR_IN_OUT, oder VAR_OUTPUT-Abschnitt deklariert wurden.

Grundsätzlich nicht visualisiert werden:

  • Arrays und Arrayelemente
  • Instanzen von Strukturen und Strukturelemente, die in dem betreffenden Funktionsbaustein deklariert wurden. Diese werden in einem separaten Display dargestellt.
  • Instanzen von Funktionsbausteinen, die in dem betreffenden Funktionsbaustein deklariert wurden. Diese werden in einem separaten Display dargestellt (siehe Instanzen von Strukturen).

Variablen, deren Werte zur Laufzeit verändert werden können, sind in blauer Farbe und unterstrichen dargestellt. Variablen mit nicht veränderbaren Werten erscheinen dagegen in normaler Schrift.

Variablen mit veränderbaren Werten sind Variablen, die in dem visualisierten Funktionsbaustein als VAR_INPUT deklariert wurden, und beim Aufruf einer Instanz dieses Funktionbausteins nicht mit Aktualparametern initialisiert werden.
Die Werte aller anderen Variablen können in ATCM Console nicht verändert werden. Nicht veränderbar sind also bspw. Werte von Variablen, die in VAR_OUTPUT-Abschnitten deklariert wurden, oder Werte von VAR_INPUT-Variablen, die beim Aufruf der Instanz des visualisierten Funktionsbausteins mit Aktualparametern initialisiert werden.

Siehe auch Variablenwerte im Display ändern.

Instanzen von Strukturen

Strukturinstanzen werden nur als Tuning Display dargestellt. Es werden alle Strukturelemente vom Typ ANY_SIMPLE dargestellt.

Grundsätzlich nicht visualisiert werden:

  • Arrays und Arrayelemente
  • Instanzen von Strukturen, die in der visulisierten Struktur deklariert werden.

Grundsätzlich sind alle im Display dargestellten Werte von Strukturinstanzen veränderbar. Die Variablen werden deshalb in blauer Farbe und unterstrichen dargestellt.

SFC-Bausteine in ATCM Console

Faceplate

Funktionsbausteine oder Programme, die im Grafik-Editor als SFC-Ablaufdiagramme programmiert wurden, werden im Faceplate analog zur Darstellung im Grafik-Editor visualisiert, d.h. die grafischen Repräsentationen, die der Grafik-Editor zur Darstellung von SFC-Bausteinen verwendet, werden auch zur Visualisierung im Faceplate genutzt. Dies betrifft auch die Position und die Verbindungslinien von SFC-Bausteinen, sodaß die grafische Repräsentation in Grafik-Editor und Faceplate weitgehend identisch ist. Eine Ausnahme bilden allerdings Aktionen, deren Rumpf im Faceplate im Gegensatz zum Grafik-Editor nicht gezeigt wird. Operationen, die in Aktionen ausgeführt werden und Datenelemente in Aktionen werden daher im Faceplate nicht dargestellt.

Wie in der Instanz-Ansicht des Grafik-Editors werden auch im Faceplate aktive Schritte in grauer Farbe dargestellt. Ebenso bietet das Kontext-Menü des Faceplates, das Sie durch Klicken mit der rechten Maustaste in den Visualisierungsbereich des Faceplates aufrufen, alle Optionen zum Beobachten und Steuern von SFC-Programmteilen, die auch der der Grafik-Editor zur Verfügung stellt. So finden Sie im Kontext-Menü einen Menüpunkt zum Aktivieren des SFC-Beobachtungsmodus und Optionen zu Steuerung von Aktionen und Transitionen.
Zusätzlich bietet Ihnen das Kontext-Menü drei Optionen zu Zoomen des Faceplate-Visualisierungsbereichs. Die aktuell aktive Auswahl des Zoomfaktors ist mit einem Häkchen markiert.

Beachten Sie, daß Sie das Kontext-Menü zum Beobachten und Steuern von SFC-Programmteilen nur dann aufrufen, wenn die sich die Position des Cursors beim Klick mit der rechten Maustaste in einem Bereich befindet, der in horizontaler Richung zwischen dem äußersten rechten und äußersten linken SFC-Element befindet. Klicken Sie dagegen außerhalb dieses Bereichs, wird stattdessen das Kontext-Menü des Internet Explorers aufgerufen.

Die Vorgehensweise bei der Arbeit im SFC-Beobachtungsmodus und mit den Optionen zum Steuern von SFC-Programmteilen im Faceplate ist identisch mit der entsprechenden Vorgehensweise im Editor von ATCM Engineering.
Sie dazu Kapitel SFC-Programmteile beobachten und steuern.

Ausschließlich SFC-Programmteile, die im Grafik-Editor programmiert wurden, können als Faceplate visualisiert werden. Im ST-Editor programmierte SFC-Programmteile werden dagegen nur als Tuning Display dargestellt.
Instanzen von SFC-Programmen, die im Grafik-Editor programmiert wurden, werden im Gegensatz zu Nicht-SFC-Programminstanzen in der Navigator-Ansicht als Hyperlink angezeigt und können genauso wie Funktionsbausteininstanzen in ATCM Console visualisiert werden. Nicht-SFC-Programminstanzen können dagegen nicht in ATCM Console visualisiert werden.

Tuning Display

Die Darstellung von im Grafik-Editor programmierten SFC-Programmteilen als Tuning Display entspricht der generellen Darstellung von Funktionsbausteinen als Tuning Display.

Variablenwerte im Display ändern

Werte von Variablen, die in den Displays mit blauer Farbe und unterstrichen dargestellt werden, können zur Laufzeit auf dem Control verändert werden.

So verändern Sie einen Variablenwert:

  1. Klicken Sie im Display auf die gewünschte Variable, um den Dialog zum Ändern einer Variable aufzurufen.

  2. Im Änderungsdialog zeigt das Feld Current den aktuellen Wert der Variable an



    Geben Sie den Wert, den Sie der Variable zuweisen wollen, im Feld New ein. Beachten Sie, daß das Format des eingegeben Formats dem Datentyp der Variablen entsprechen muß. Wenn die Variable bspw. dem Datentyp LREAL angehört, muß der Variablenwert in Punktschreibweise erfolgen. Am einfachsten orientieren Sie sich bei der Eingabe an der Schreibweise des aktuellen Werts im Feld Current.



    Beim Ändern des Werts einer Variablen vom Typ BOOL erscheinen im Änderungsdialog statt dem Eingabefeld zwei runde Optionsfelder, über die Sie per Mausklick den neuen Wert festlegen können. In diesem Fall ergibt sich das korrekte Eingabeformat natürlich automatisch.

  3. Wenn das Format des von Ihnen eingegebenen Werts dem Datentyp der Variable entspricht, wird die Schaltfläche Write aktiviert. Klicken Sie auf diese Schaltfläche, um die Variable mit dem neuen Wert zu aktualisieren.

  4. Um den Dialog zu schließen, klicken Sie auf Close.
