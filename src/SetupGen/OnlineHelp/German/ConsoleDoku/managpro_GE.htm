Dieses Kapitel beschreibt, wie Sie Projekte verwalten können. Sie lernen, wie Sie ein Projekt erstellen, öffnen, schließen, speichern, drucken und löschen, und wie Sie es generieren, hinunterladen und debuggen.

Erstellen eines neuen Projekts

Zur Erstellung eines neuen Projekts gehen Sie in das Menü File, und klicken Sie auf New Project,

- ODER -

klicken Sie auf die Schaltfläche New Project in der Werkzeugleiste.

Folgen Sie den Anweisungen des Wizard, um ein neues Projekt zu erstellen.

Öffnen eines Projekts

Um ein Projekt zu öffnen, gehen Sie in das Menü File, und klicken Sie auf Open Project

- ODER -

klicken Sie auf die Schaltfläche Open Project auf der Werkzeugleiste.

Schließen eines Projekts

Wenn Sie das aktuelle Projekt schließen wollen, gehen Sie in das Menü File und klicken Sie auf Close Project.
Das Projekt wird geschlossen.

Speichern des aktuellen Projekts

Speichern

Zum Speichern des aktuellen Projekts gehen Sie in das Menü File und klicken auf Save Project,

- ODER -

klicken Sie auf die Schaltfläche Save Project in der Werkzeugleiste.
Das Projekt wird gespeichert.

Löschen des aktuellen Projekts

Bevor Sie ein Projekt löschen können, müssen Sie es öffnen.
Klicken Sie dann im Menü File auf Delete Project. Quittieren Sie die Sicherheitsabfrage mit Yes, um das Projekt zu löschen.

Dabei werden alle Dateien dieses Projekts gelöscht!

Drucken des Projekts

Diese Funktion druckt alle Dateien Ihres aktuellen Projekts aus.

Druckereinrichtung

Vor dem ersten Druckvorgang müssen Sie den Drucker einrichten.

  1. Klicken Sie im Menü File auf Print Setup... .
    Der Setup-Dialog wird eingeblendet.



  2. Geben Sie alle erforderlichen Informationen ein und bestätigen Sie mit OK.
    Weitere Einzelheiten finden Sie im Handbuch Ihres Druckers.

Drucken

Um das Projekt auszudrucken, gehen Sie in das Menü File und klicken auf Print Project,

- ODER -

klicken Sie auf die Schaltfläche Print Project auf der Werkzeugleiste.
Das Projekt wird ausgedruckt.

Generieren des Projekts

Sie müssen jede Konfiguration (Standardvorgabe: Configuration.CON) einem PC zuweisen, bevor Sie das Projekt generieren können.
  1. Klicken Sie auf den Hardware-Baum in der Navigator-Ansicht.
  2. Klicken Sie mit der rechten Maustaste auf das 4CProject-Symbol, um das Kontext-Menü einzublenden.
  3. Wählen Sie Insert PC... .
    Der Dialog Add PC erscheint.
  4. Geben Sie den Namen des PC ein (sehen sie ggf. in in der NT-Systemsteuerung unter Netzwerk nach), und bestätigen Sie mit OK.
  5. Gehen Sie auf das Symbol des gewünschten PC.
  6. Klicken Sie mit der rechten Maustaste darauf, um das Kontext-Menü zu öffnen.
  7. Wählen Sie Assignment... .
    Der Dialog Assignment erscheint.
  8. Wählen Sie die Konfiguration, die Sie dem PC zuweisen wollen, und klicken Sie auf OK.
    Diese Konfiguration ist nun für das weitere Vorgehen vorbereitet.

Der Aufwand bei der Generierung des Codes mit ATCM Engineering ist stark rationalisiert. Wenn Sie sich alle Fehlermeldungen anzeigen lassen wollen,

gehen Sie in das Menü Build und klicken Sie auf Build All,

- ODER -

klicken Sie auf die Schaltfläche Build All der Werkzeugleiste.

Die Message-Ansicht erscheint im unteren Teil des ATCM Engineering-Fensters; hier werden alle Fehlermeldungen und Warnhinweise angezeigt.
Für einen Download muß das Projekt fehlerfrei compiliert worden sein.

Download der Konfiguration des aktuellen Projekts

Sobald Sie den Code generiert haben, können Sie ihn ganz einfach an ATCM Control hinunterladen. Gehen Sie hierzu wie im folgenden beschrieben vor.
Je nachdem wie viele Konfigurationen zum aktuellen Projekt gehören, und wie viele gleichzeitig an die PCs hinuntergeladen werden sollen, können Sie die Konfigurationen an die jeweiligen ATCM Controls hinunterladen oder aktualisieren.

Beim Compilieren des Projekts generiert der Compiler Domänen von jedem Objekt. Diese Domänen sind Datenpakete, die Informationen über den Quellcode und über die Visualisierung des Objekts enthalten. Die Domänen werden im Dialog Selective Update angezeigt.

In der Standardeinstellung wird unmittelbar nach dem Download die Konfiguration gestartet.

Download 

Alle Konfigurationen

Wenn Sie alle Konfigurationen in einem einzigen Schritt an alle ATCM Controls hinunterladen wollen,

gehen Sie in das Menü Build und klicken Sie auf Download/All Configurations,

- ODER -

klicken Sie auf die Schaltfläche All Configurations auf der Werkzeugleiste.

Falls noch keine Debug-Verbindung zum ATCM Control besteht, erscheint auf Ihrem Bildschirm ein Dialogfeld mit der Frage, ob eine Debug-Verbindung hergestellt werden soll.
Während einer Debugverbindung ist nur einem ATCM Engineering der Zugang zum ATCM Control erlaubt.

Klicken Sie auf Yes, wenn Sie ein neues Projekt hinunterladen möchten.

ATCM Engineering beginnt jetzt mit dem Aufbau einer Debug-Verbindung.

Sie können den Verbindungsaufbau abbrechen, indem Sie auf die Schaltfläche Cancel klicken.

Falls die Debug-Verbindung nicht hergestellt werden kann, weil bspw. bereits ein anderer Anwender eine Debug-Verbindung eingerichtet hat, erscheint nach einiger Zeit die folgende Meldung:

Klicken Sie in diesem Fall auf Yes, um die bereits bestehende Verbindung zu beenden.

Sie können den Timeout, d.h. die Zeitdauer, während der ein Verbindungsaufbau versucht wird, bevor eine Fehlermeldung erscheint, in der Registerkarte des Dialogfelds Options ändern. Das Dialogfeld Options rufen Sie auf, indem Sie im Menü Tools auf Project Settings klicken.

Nachdem eine Debug-Verbindung erfolgreich hergestellt wurde, wird der Steuerungscode aller ATCM Controls gelöscht, und alle Konfigurationen des aktuellen Projekts werden an alle ATCM Controls gleichzeitig hinuntergeladen.

Ausgewählte Konfigurationen

Um eine einzelne Konfiguration an das zugewiesene ATCM Control hinunterzuladen, gehen Sie in das Menü Build, klicken dort auf Download und wählen im Menü das jeweilige ATCM Control an. Die Message-Ansicht unten im Bildschirm wechselt auf die Registerkarte Download und zeigt die Steuerungsmeldungen an.

Der Steuerungscode des ATCM Control wird gelöscht und die zugewiesene Konfiguration an das gewählte ATCM Control hinuntergeladen.

Download Changes

Alle Konfigurationen

Wenn Sie die Unterschiede zwischen dem Code des laufenden ATCM Controls und dem aktuellen Stand des Quellcodes auf Ihrem ATCM Engineering hinunterladen möchten, können Sie die laufenden ATCM Controls aktualisieren.
Im Gegensatz zu Download All berücksichtigt diese Funktion nur die Unterschiede zwischen den Daten auf ATCM Control und dem aktuellen Projekt auf Ihrem ATCM Engineering. Nach der Generierung wird ATCM Control lediglich mit den Unterschieden aktualisiert. Dadurch sparen Sie beim Download Zeit. In diesem Fall werden alle geänderten oder neuen Daten an alle ATCM Controls übergeben.
Gehen Sie dazu in das Menü Build und klicken Sie auf Download Changes / All Configurations.
Falls noch keine Debug-Verbindung zum ATCM Control besteht, erscheint auf Ihrem Bildschirm ein Dialogfeld mit der Frage, ob eine Debug-Verbindung hergestellt werden soll.
Während einer Debugverbindung ist nur einem ATCM Engineering der Zugang zum ATCM Control erlaubt.

Klicken Sie auf Yes, wenn Sie ein neues Projekt hinunterladen möchten.

ATCM Engineering beginnt jetzt mit dem Aufbau einer Debug-Verbindung.

Sie können den Verbindungsaufbau abbrechen, indem Sie auf die Schaltfläche Cancel klicken.

Nach einem erfolgreichen Verbindungsaufbau wird sofort die Aktualisierung gestartet.

Ausgewählte Konfigurationen

Wenn Sie die Unterschiede zwischen dem Laufzeitcode eines einzelnen ATCM Controls und dem aktuellen Stand des Projekts auf Ihrem ATCM Engineering hinunterladen möchten, können Sie jedes laufende ATCM Control einzeln aktualisieren.
Im Gegensatz zur Option Download/All Configurations berücksichtigt diese Funktion nur die Unterschiede zwischen den Daten auf dem gewählten ATCM Control und dem aktuellen Projekt auf Ihrem ATCM Engineering. Nach der Generierung wird ATCM Control lediglich mit den Unterschieden aktualisiert. Dadurch sparen Sie beim Download Zeit. In diesem Fall werden die geänderten oder neuen Daten an das gewählte ATCM Control übergeben.
Gehen Sie dazu in das Menü Build und klicken Sie unter Download Changes / Selected Configurations auf das gewünschte ATCM Control.

Der Aktualisierungsvorgang beginnt unmittelbar danach.

Download Selection

Konfigurations-Übersicht

Im Dialogfeld Download Overview können Sie den Download-Status aller ATCM Controls des Projekts überwachen. Um das Dialogfeld aufzurufen, gehen Sie in das Menü Build und klicken auf Configuration Overview. Alle ATCM Controls werden nach Name aufgelistet. Sie erhalten Informationen über den Namen des hinuntergeladenen Projekts, das letzte Download, den Benutzer, der das Projekt zuletzt hinuntergeladen hatte, sowie den aktuellen Status.

Sie können sich die Unterschiede zwischen den Daten des auf Ihrem ATCM Engineering befindlichen Projekts und dem Inhalt von ATCM Control ansehen, indem Sie das gewünschte ATCM Control wählen und auf Details klicken.
Der Dialog Selective Update erscheint.

Selektive Aktualisierung

Alle Domänen des Projekts werden nach Name und Typ aufgelistet.
ATCMControl vergleicht alle Domänen auf ATCM Control mit allen Domänen des aktuellen Projekts auf Ihrem ATCM Engineering. Die Unterschiede werden in diesem Dialog aufgeführt. In der Spalte Original Action to do erhalten Sie Informationen über den gegenwärtig vorgeschlagenen Status jeder Domäne auf ATCM Control. Bei diesem Status handelt es sich nur um einen Vorschlag für jede Domäne. Es ist jedoch die Aktion, die ATCMControl als nächste bei jeder Domäne durchführen würde. Wenn Sie auf Start Download klicken, laden Sie alle Domänen des Projekts so an ATCM Control hinunter, wie sie vorgeschlagen wurden.
Sie können diese Vorschläge aber auch ändern. Bestimmen Sie im einzelnen, was mit der gewählten Domäne passieren soll.
Klicken Sie dazu mit der rechten Maustaste auf die gewünschte Domäne.

Mehrfachmarkierungen sind möglich! Halten Sie die Umschalttaste gedrückt, und wählen Sie die gewünschten Domänen.
Bei einer Mehrfachauswahl von Domänen bezieht sich das Kontext-Menü, das beim Klicken mit der rechten Maustaste eingeblendet wird, auf alle gewählten Domänen.

Debuggen des Projekts

Das Debuggen eines Projekts hilft Ihnen Fehler in Ihrem Code zu finden. Sie können zum Debuggen Breakpoints einsetzten und Code im sogenannten Single Step Run Zeile für Zeile abarbeiten.

Der Einsatz von Breakpoints und Single Step Run ist sowohl im Text- als auch im graphischen Editor möglich.

Sie können Ihren Code nur dann debuggen, wenn eine Debug-Verbindung hergestellt wurde. Während einer Debug-Verbindung ist nur die Verbindung von einem ATCM Engineering zum ATCM Control erlaubt. Um eine Debug-Verbindung herzustellen, klicken Sie im Menü Control auf Enable Debug und wählen die gewünschte Konfiguration (den PC) aus.

Einfaches / erweitertes Debuggen

Beachten Sie bitte, daß intensives Debuggen die Ausführungszeit Ihres Codes beeinträchtigt. Aus diesem Grund stehen in ATCMControl zwei Debug-Modi zur Verfügung. Den Debug-Modus können Sie in der Registerkarte Debug im Dialogfeld Options festlegen. Sie rufen dieses Dialogfeld über die Option Project Settings im Menü Tools auf.

Nachdem Sie den Debug-Modus geändert haben, sollten Sie das Projekt neu kompilieren und herunterladen.

Während der Modus Simple Debugging das Laufzeitverhalten des Controls nicht merklich beeinflußt, verringert der Modus Advanced Debugging die Systemleistung um c.a. 30 %. Folgende Unterschiede bestehen zwischen den beiden Debug-Modi:

  • Simple Debugging: Breakpoints sind am Beginn oder Ende eines Programms oder Funktionsbausteins erlaubt. Im ST-Code sind diese Stellen im Code durch die erste Anweisung (d.h. die erste Linie im Code, die mit einem Semikolon nach der Variablendeklaration abgeschlossen wird) und der END_PROGRAM- oder END_FUNCTION_BLOCK-Anweisung definiert. Im FBD-Editor sind spezielle Stellen für Beginn oder Ende eines Programms oder Funktionsbausteins in der oberen linken und der unteren rechten Ecke definiert.



  • Advanced Debugging: Breakpoints sind in diesem Modus in allen Anweisungen erlaubt. Da dieser Debug-Modus die System-Ressourcen stark in Anspruch nimmt, verringert sich die Systemleistung um c.a. 30 %. Beim Compilieren eines Projekts, dessen Debug-Modus auf Advanced Debugging gesetzt wurde, wird daher in der Message-Ansicht eine Meldung ausgegeben, daß das Projekt mit erweiterten Debugeinstellungen compiliert wird.

Breakpoint setzen/entfernen

Breakpoints können zu Programminstanzen und Funktionsbausteinen gesetzt werden.
Um einen Breakpoint zu setzen,

positioneren Sie den Cursor im Quellcode-Editor an die Stelle, an der Sie den Breakpoint setzen möchten und klicken im Kontext-Menü auf Set/Remove Breakpoint,

- ODER -

positioneren Sie den Cursor im Quellcode-Editor an die Stelle an der Sie den Breakpoint setzen möchten, und drücken Sie auf die F9-Taste.

Der Breakpoint wird im Editor an die nächste mögliche Stelle in der Nähe der aktuellen Cursorposition gesetzt. Sie können einen gesetzten Breakpoint an dem roten Kreis am Rand des Editors erkennen.

Um einen Breakpoint zu entfernen

setzen Sie den Cursor im Editor in die Nähe des Breakpoints, den Sie entfernen möchten und klicken im Kontext-Menü auf Set/Remove Breakpoint,

- ODER -

setzen Sie den Cursor im Editor in die Nähe des Breakpoints, den Sie entfernen möchten und drücken Sie die F9-Taste.

 

Um alle Breakpoints zu entfernen

klicken Sie im Kontext-Menü auf Remove All Breakpoints,

- ODER -

drücken Sie die Tastenkombination Crtl-Shift-F9.

Arbeiten mit Breakpoints

Breakpoints werden in den Editoren mit den folgenden Symbolen dargestellt:

Aktivierter Breakpoint, bevor die Programmausführung an dieser Stelle gestoppt wird.
Aktivierter Breakpoint, an dem die Programmausführung gestoppt wurde.
Deaktivierter Breakpoint, der beim Debuggen ignoriert wird.

Um einen Breakpoint zu aktivieren oder zu deaktivieren klicken Sie auf Enable/Disable Breakpoint im Kontext-Menü,

- ODER -

drücken Sie die Tastenkombination Strg-F9.

Nach dem Erreichen eines Breakpoints geht der laufende Task in den sogenannten Break-Status über. Um die Programmausführung fortzusetzen, existieren zwei Möglichkeiten:

Programmausführung bis zum nächsten Breakpoint fortsetzen. Klicken Sie dazu im Kontext-Menü auf Continue, oder drücken Sie die F5-Taste.

Programm bis zur nächsten Stelle ausführen, an der ein Breakpoint erlaubt ist (diese Stelle ist abhängig davon ob fortgeschrittenes oder einfaches Debuggen eingestellt ist). Klicken Sie dazu auf Single Step im Kontext-Menü oder drücken Sie die F10-Taste.

Beenden von ATCM Engineering

Zum Beenden von ATCM Engineering gehen Sie in das Menü File und klicken auf Exit.

Alle nicht gespeicherten Änderungen werden gespeichert, ATCM Engineering wird beendet, und alle Verbindungen zu ATCM Control werden unterbrochen.
