F_SYNC_ARRAYWRITEFILE

Writes data from an array of bytes into  a file.

Interface

FUNCTION F_SYNC_ARRAYWRITEFILE : BOOL

VAR_INPUT
    DATA        : ARRAY OF BYTE;
    POS         : DINT;
    LENGTH      : DINT;
END_VAR

VAR_IN_OUT
    FILE        : F_SYNC_FILE;
END_VAR

Return value

Value of the type BOOL
Signalizes the error status.
Possible values:
TRUE: An error has occurred.
FALSE: No error has occurred.

Parameter

INPUT


DATA
A variable of type ARRAY[...] OF BYTE.
Contains the write data.
POS
Value of type DINT.
Position in array where to store data.
It is relative to lower bound 0 even if the array is dimensioned with a lower bound not equal 0.
LENGTH
Value of type DINT.
Number of bytes to write.

IN_OUT

FILE
Instance of the F_SYNC_FILE structure
File object on which the operation is performed.

Description

F_SYNC_ARRAYWRITEFILE writes the value passed to DATA in the file passed to FILE. Before writing in a file, the file must be opened with F_SYNC_OPENFILE. To allow write access, the access mode must be set to CF_READ_WRITE or CF_APPEND in the MODE parameter of the structure instance of F_SYNC_FILE passed to F_SYNC_OPENFILE before you open the file.

The file pointer which indicates the current write position is represented by the structure variable POSITION of the structure instance of F_SYNC_FILE passed to FILE. The first write access following the opening of a file is always performed at the position defined by the access mode unless the file pointer has been explicitly set to a different position (see below).

After each write access, the value of POSITION is automatically set to a position directly after the last byte written. This allows you to write sequentially in a file without having to explicitly set the file pointer to the new position.

The position of a write access can be explicitly set by assigning a value to the structure variable POSITION. For this purpose, the value of POSITION must be set to the desired writing position after opening a file and before the write access. On conclusion of the writing operation, the value of POSITION is again set to the position directly after the last byte written.

To convert data of type ANY_TYPE into an ARRAY OF BYTE you can use the COPY_XXXX_TO_BYTE ARRAY functions located in the ATCM Utility library.
