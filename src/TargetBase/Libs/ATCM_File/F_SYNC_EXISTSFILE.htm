F_SYNC_EXISTSFILE

Checks whether a file exists.

Interface

FUNCTION F_SYNC_EXISTSFILE:BOOL
VAR_IN_OUT
    FILE        : F_SYNC_FILE;
END_VAR

Return value

Value of the type BOOL
Signalizes if the file exits.
Possible values:
TRUE: File exists.
FALSE: File does not exist.

Parameter

IN_OUT

FILE
Instance of the F_SYNC_FILE structure
In contrast to most of the other functions provided in the group of blocks for elementary synchronous file operations, the structure instance passed to FILE does not represent a file object on which the corresponding file operation is performed. The FILENAME element of the F_SYNC_FILE structure is used instead for defining the name of the file to be found.

Description

F_SYNC_EXISTSFILE checks whether a file exists.

The name of the file to be found is passed to F_SYNC_EXISTSFILE in the parameter FILE. FILE expects an instance of the F_SYNC_FILE structure. This structure includes the structure element FILENAME which specifies a file name and must be initialized before the structure instance is passed to F_SYNC_EXISTSFILE .

The call of F_SYNC_EXISTSFILE is independent of an access cycle, which means that a previous call of F_SYNC_OPENFILE is not required.
