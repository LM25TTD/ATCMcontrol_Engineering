F_SYNC_OPENFILE

Opens a file for editing.

Interface

FUNCTION F_SYNC_OPENFILE:BOOL
VAR_IN_OUT
    FILE        : F_SYNC_FILE;
END_VAR

Return value

Value of the type BOOL
Signalizes the error status.
Possible values:
TRUE: An error has occurred.
FALSE: No error has occurred.

Parameter

IN_OUT

FILE
Instance of the F_SYNC_FILE structure.
File object on which the operation is performed.

Description

F_SYNC_OPENFILE opens the file passed to FILE for read or write access. F_SYNC_OPENFILE initiates an access cycle and therefore must always be called before a read or write access can be executed via other function block instances of the group of blocks for elementary synchronous file operations.

The file to be opened is represented by an instance of the F_SYNC_FILE structure. This structure contains the MODE element with which you define the access mode a file is to be opened with. The possible values are defined in the access mode constants.

Before you open a file, you need to set the access mode in the structure instance of F_SYNC_FILE passed to F_SYNC_OPENFILE. The access mode set during opening remains in effect until the file is closed again with F_SYNC_CLOSEFILE. All functions which are called between the opening and closing of a file and which work on the file object passed to F_SYNC_OPENFILE are subject to the access authorizations that have been set in MODE when opening the file.

When a file is opened, the position of th file pointer always depends on the access mode, which is set in the instance of F_SYNC_FILE passed to F_SYNC_OPENFILE. When opening a file with F_SYNC_OPENFILE, an initialization of the element variable POSITION of the F_SYNC_FILE structure with a user-defined value is overwritten by the value that is assigned to the corresponding access mode. If a file access is to be executed at a specific position after having opened the file, you therefore need to explicitly set the value of POSITION anew after opening.
