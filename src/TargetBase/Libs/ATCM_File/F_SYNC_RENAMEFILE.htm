F_SYNC_RENAMEFILE

Renames a file or a directory.

Interface

FUNCTION F_SYNC_RENAMEFILE:BOOL
VAR_INPUT
    NEWNAME     : STRING;
END_VAR
VAR_IN_OUT
    FILE        : F_SYNC_FILE;
END_VAR

Return value

Value of the type BOOL
Signalizes the error status.
Possible values:
TRUE: An error has occurred.
FALSE: No error has occurred.

Parameter

INPUT


NEWNAME
Value of the type STRING
Name to which the file is to be renamed.
The file name must be indicated by a fully qualified path (for example, /control/Test.txt). If no path is specified, the final position is undefined.
If the entered path differs from the path specified in the parameter FILENAME of the structure instance of F_SYNC_FILE which is passed to FILE, the renamed file is moved to the directory indicated in NEWNAME.
If the file specified in NEWNAME already exists, the operation is aborted and an error is output.
The compiler does not check whether the syntax of the file name and path information is correct or whether they are plausible.

IN_OUT

FILE
Instance of the F_SYNC_FILE structure
File object on which the operation is performed.

Description

F_SYNC_RENAMEFILE renames a file the name specified in the parameter NEWNAME. The file to be renamed is given in the parameter FILENAME of the instance of F_SYNC_FILE passed to F_SYNC_RENAMEFILE. After the file has been renamed, this parameter receives the new file name.

F_SYNC_RENAMEFILE constitutes an exception among the group of blocks for elementary synchronous file operations. In contrast to most other functions of this group, an instance of F_SYNC_RENAMEFILE must not be called within an access cycle since only files that are currently not accessed by another process may be renamed. The file passed to F_SYNC_RENAMEFILE therefore must not be currently open due to F_SYNC_OPENFILE or due to an external program.
