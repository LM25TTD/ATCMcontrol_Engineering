Function Name

modbus_set_slave - set slave number in the context

Interface

FUNCTION MODBUS_SET_SLAVE : UINT

VAR_INPUT

ctx : STRING;

slave : BYTE;

END_VAR

Return value

Value of type UINT

0 of success 1 for ERROR

Parameters

INPUT

ctx

Value of type STRING

Context for the connection to be closed. Valid values are RTU, TCP, TCPRTU

slave

Value of type BYTE

Slave id to be set. Valid value are 1-254

Description

The MODBUS_SET_SLAVE function shall set the slave number in the libmodbus context. The behavior depends of network and the role of the device:

*RTU* or *TCPRTU* Define the slave ID of the remote device to talk in master mode or set the internal slave ID in slave mode. According to the protocol, a Modbus device must only accept message holing its slave number or the special broadcast number.

*TCP* The slave number is only required in TCP if the message must reach a device on a serial network. The special value 'MODBUS_TCP_SLAVE' (0xFF) can be used in TCP mode to restore the default value.

The broadcast address is 'MODBUS_BROADCAST_ADDRESS' (0x0). This special value must be use when you want all Modbus devices of the network receive the request.
